# 课伴系统架构设计文档

> 版本：v1.0 | 最后更新：2026-07-08

---

## 一、架构理念

课伴采用**本地优先（Local-First）+ AI增强可选**的双层架构。

核心思想：**所有核心功能完全运行在客户端本地**，不依赖网络连接或云端服务；AI能力作为可选插件按需加载，网络恢复时自动开启同步，断网时无感回退到纯本地模式。

这一理念确保：
- 用户在飞机上、地铁里、偏远地区均可完整使用所有学习功能
- 数据主权完全归属于用户，隐私零风险
- AI功能锦上添花，而非必要条件

## 三、关键设计原则

### 3.1 本地核心独立

> 客户端功能模块仅依赖本地存储抽象层，不直接调用任何网络API。

- 每个功能模块（番茄钟、笔记、闪卡、费曼）的写操作只写入本地存储
- 所有网络通信（同步、AI调用）由独立层统一处理，模块层完全无感知
- 确保任何模块可独立测试，不依赖网络环境
- 模块间零耦合，通过事件总线（EventBus）进行松耦合通信

### 3.2 AI插件化

> AI能力作为可选插件按需加载，不加载时零性能开销。

- AI插件实现统一接口 `AIPlugin`，各功能模块通过接口调用
- 插件通过动态 `import()` 按需加载，未加载时不占用任何资源
- 提供 `LocalFallback` 实现：AI不可用时返回友好提示，不报错
- UI中AI功能按钮始终显示，点击时检测可用性再决定是否调用

### 3.3 渐进式增强

> 网络可用时自动开启同步和AI功能，网络断开时无感回退到纯本地。

- Service Worker 监听网络状态变化，自动切换在线/离线模式
- 在线时：操作日志实时推送到同步服务，AI插件可直接调用云端模型
- 离线时：操作进入本地队列，网络恢复后自动批量重放
- 状态栏显示当前连接状态，但不阻断任何用户操作

## 四、客户端架构详细设计

### 4.1 功能模块层

#### 番茄钟模块（PomodoroModule）

**职责边界：**
- 管理计时器生命周期（正计时/倒计时/暂停/重置）
- 支持专注模式、休息模式、自定义模式切换
- 记录每次番茄钟会话到本地存储
- 统计面板：今日专注时长、本周趋势图、热力图

**对外接口：**
```typescript
interface PomodoroModule {
  start(mode: 'focus' | 'break' | 'custom', duration?: number): void
  pause(): void
  resume(): void
  reset(): void
  getStats(range: DateRange): PomodoroStats
  onSessionComplete(cb: (session: PomodoroSession) => void): Unsubscribe
}
```

#### 智能笔记模块（NoteModule）

**职责边界：**
- 富文本编辑（基于TipTap），支持Markdown快捷输入
- 模板系统：康奈尔笔记、思维导图、大纲笔记等预设模板
- 标签和文件夹两级组织体系
- AI摘要功能（插件化，可选）

**对外接口：**
```typescript
interface NoteModule {
  createNote(template?: TemplateType): Note
  updateNote(id: string, patch: Partial<Note>): void
  deleteNote(id: string): void
  searchNotes(query: SearchQuery): Note[]
  summarizeNote(id: string): Promise<string>  // 依赖AI插件
  exportNote(id: string, format: 'md' | 'pdf' | 'html'): Blob
}
```

#### 闪卡模块（FlashcardModule）

**职责边界：**
- 基于SM-2算法的间隔重复调度
- 牌组（Deck）管理：创建、分类、导入导出
- 复习界面：卡片翻转、评分（1-4分）
- AI自动生成闪卡（插件化，可选）

**对外接口：**
```typescript
interface FlashcardModule {
  getDueCards(deckId?: string): Flashcard[]
  reviewCard(cardId: string, quality: 1|2|3|4): void
  createCard(deckId: string, front: string, back: string): Flashcard
  generateCards(noteId: string): Promise<Flashcard[]>  // 依赖AI插件
  importDeck(file: File): Deck
  exportDeck(deckId: string): Blob
}
```

#### 费曼学习模块（FeynmanModule）

**职责边界：**
- 概念录入与讲解文本记录
- 薄弱点自动/手动标注
- AI评估讲解质量（插件化，可选）
- 学习进度追踪

**对外接口：**
```typescript
interface FeynmanModule {
  createSession(concept: string): FeynmanSession
  updateExplanation(sessionId: string, text: string): void
  markWeakPoint(sessionId: string, section: string): void
  evaluateSession(sessionId: string): Promise<EvaluationResult>  // 依赖AI插件
  getHistory(filter?: FilterOptions): FeynmanSession[]
}
```

### 4.2 存储抽象层（Storage Adapter）

**设计目标：** 功能模块层通过统一接口访问数据，不感知底层存储实现。

```typescript
interface StorageAdapter {
  // CRUD基础操作
  get<T>(store: string, id: string): Promise<T>
  getAll<T>(store: string, query?: QueryOptions): Promise<T[]>
  put<T>(store: string, item: T): Promise<string>
  delete(store: string, id: string): Promise<void>
  
  // 批量操作
  batch<T>(operations: BatchOperation<T>[]): Promise<void>
  
  // 查询能力
  where<T>(store: string, index: string, value: any): Promise<T[]>
  count(store: string, query?: QueryOptions): Promise<number>
  
  // 事务支持
  transaction<T>(stores: string[], fn: () => Promise<T>): Promise<T>
  
  // Schema迁移
  migrate(version: number, upgrader: MigrationUpgrader): void
  
  // 数据导入导出
  exportData(stores: string[], format: 'json' | 'csv'): Promise<Blob>
  importData(blob: Blob, strategy: 'merge' | 'replace'): Promise<void>
}
```

### 4.3 Service Worker 层

**离线缓存策略：**

| 资源类型 | 缓存策略 | 说明 |
|---------|---------|------|
| 静态资源（JS/CSS/图片） | Cache First | 优先读缓存，后台更新 |
| HTML页面 | Network First + Cache Fallback | 优先网络，离线返回缓存 |
| API数据 | Stale While Revalidate | 返回旧数据同时后台刷新 |
| AI响应 | Cache First + TTL | 缓存7天，过期重新请求 |

**后台同步队列：**
```
离线操作 → 写入本地队列(IndexedDB)
         → Service Worker监听sync事件
         → 网络恢复后批量重放队列
         → 失败操作重试(指数退避,最多3次)
         → 最终失败的操作标记为冲突，等待用户处理
```

### 4.4 AI插件层

**接口定义：**
```typescript
interface AIPlugin {
  isAvailable(): boolean
  summarize(text: string): Promise<string>
  generateCards(note: string): Promise<Flashcard[]>
  evaluateExplanation(text: string): Promise<EvaluationResult>
  recommendDuration(history: PomodoroHistory[]): Promise<DurationConfig>
}

class LocalFallback implements AIPlugin {
  isAvailable(): boolean { return false }
  async summarize(): Promise<string> {
    throw new AINotAvailableError('连接网络后可使用智能摘要功能')
  }
  // ... 其他方法同理
}
```

**加载策略：**
- 应用启动时不加载AI插件，零开销
- 用户首次点击AI功能按钮时，动态 `import()` 加载插件模块
- 加载成功后缓存插件实例，后续调用直接使用
- 加载失败或网络不可用，注入 `LocalFallback` 实例

## 五、云端架构详细设计（MVP-2）

### 5.1 同步服务端架构

```
客户端 → Nginx Ingress → Go Sync Service → PostgreSQL
                                         → Redis (会话/临时数据)
```

**核心组件：**
- **操作日志接收器**：接收客户端批量推送的操作日志，验证后入库
- **冲突检测引擎**：基于版本向量比对，识别真正冲突并标记
- **增量推送服务**：仅推送客户端未同步的变更，减少流量
- **WebSocket通道**：在线时实时双向同步，离线时降级为HTTP轮询

**技术栈：** Go + Gin + GORM + PostgreSQL + Redis

---

### 5.2 AI服务网关架构

```
AI插件 → HTTPS → FastAPI Gateway → LangChain编排 → LLM API（OpenAI/本地模型）
                                  → Prompt模板库
                                  → 调用频率限制器
                                  → 结果缓存层(Redis)
```

**核心设计：**
- **模型路由**：根据任务类型自动选择最优模型（摘要→轻量模型，生成→大模型）
- **Prompt模板管理**：版本化管理，支持A/B测试
- **调用频率限制**：用户级+全局级双层限流，防止滥用
- **结果缓存**：相同输入7天内返回缓存结果，节省API调用成本

---

### 5.3 云账户系统

**功能范围：**
- 用户注册/登录（邮箱+OAuth2第三方登录）
- JWT Token管理（AccessToken 15min + RefreshToken 7d）
- 设备绑定与解绑（最多5台设备）
- 数据导出（完整个人数据ZIP包）
- 账户注销（30天冷静期后彻底删除所有数据）

## 六、数据流设计

### 6.1 本地数据流

```
用户操作（点击/输入）
    ↓
功能模块（业务逻辑处理）
    ↓
存储抽象层（统一接口调用）
    ↓
Dexie.js（IndexedDB封装）
    ↓
IndexedDB（浏览器本地持久化）
    ↓
UI响应更新（通过Zustand状态订阅）
```

**写操作路径（以创建闪卡为例）：**
1. 用户在闪卡模块点击"新建卡片"
2. `FlashcardModule.createCard()` 调用 `StorageAdapter.put('flashcards', card)`
3. `StorageAdapter` 调用 `Dexie.table('flashcards').add(card)`
4. 写操作同时写入 `operation_log` 表（用于后续同步）
5. Zustand状态更新，UI重新渲染卡片列表

---

### 6.2 同步数据流

```
网络恢复
    ↓
Service Worker sync事件触发
    ↓
读取本地操作日志队列（IndexedDB）
    ↓
批量POST /api/sync/batch 到服务端
    ↓
服务端比对版本向量
    ├─ 无冲突：合并更新，返回最新版本号
    └─ 有冲突：标记冲突记录，返回双方版本
                ↓
        客户端UI提示用户选择保留版本
                ↓
        用户选择后，提交冲突解决结果
```

---

### 6.3 AI数据流

```
用户点击AI功能按钮（如"智能摘要"）
    ↓
检测AI插件是否已加载
├─ 未加载：动态import()加载插件
│   ├─ 加载成功：继续
│   └─ 加载失败：显示"连接网络后可使用智能功能"
└─ 已加载：继续
    ↓
调用 AIPlugin.summarize(text)
    ↓
插件发送HTTPS请求到AI服务网关
    ↓
FastAPI Gateway → LangChain → LLM API
    ↓
返回摘要结果
    ↓
插件将结果回传给功能模块
    ↓
模块渲染摘要内容到UI
```
