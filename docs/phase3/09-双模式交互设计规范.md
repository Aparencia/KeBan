# 课伴双模式交互设计规范

> 版本：v1.0 | 阶段：Phase 3 | 最后更新：2026-07-08

---

## 一、设计原则

### 1.1 核心理念

课伴采用**"功能级按需激活"**设计范式，而非传统的"全局二元切换"模式。这意味着：

- **不存在"本地模式"与"AI模式"的全局切换开关**
- 用户无需理解底层技术架构（本地 vs 云端、AI vs 非AI）
- 用户只需感知：**核心功能随时可用，联网时有额外智能功能**

### 1.2 设计原则清单

| 编号 | 原则 | 说明 |
|------|------|------|
| P1 | 透明性 | 网络状态和AI可用性对用户透明，但不强推 |
| P2 | 渐进增强 | 离线=完整基础功能，在线=基础+智能增强 |
| P3 | 零打扰 | 状态变化不打断用户当前工作流 |
| P4 | 可预测性 | 相同操作在不同网络状态下行为一致或优雅降级 |
| P5 | 即时反馈 | 所有操作300ms内给出视觉反馈 |
| P6 | 容错设计 | 网络波动不导致数据丢失或操作失败 |

### 1.3 反模式（禁止事项）

- ❌ 禁止全局模式切换弹窗（如"是否切换到AI模式？"）
- ❌ 禁止离线时显示大面积功能锁定遮罩
- ❌ 禁止网络断开时弹出警告对话框
- ❌ 禁止强制用户登录才能使用基础功能
- ❌ 禁止AI功能不可用时显示灰色不可点击按钮（应保持可点击+友好提示）

## 二、网络状态指示器设计

### 2.1 全局状态指示器

位于应用顶部导航栏右侧（桌面端）或底部状态栏（移动端），设计为微小且不干扰的视觉元素。

#### 状态定义

| 状态 | 视觉表现 | 说明 | 用户感知 |
|------|----------|------|----------|
| 🟢 在线+AI可用 | 小绿点（8px）+ "已连接"标签 | 网络正常且AI服务可用 | 所有功能可用 |
| 🟡 在线（无AI） | 小黄点（8px）+ "已连接"标签 | 网络正常但AI服务不可用 | 基础功能+同步可用 |
| ⚪ 离线 | 无指示器（默认状态） | 无网络连接 | 不提示离线，静默降级 |

#### 视觉规范

```
位置：导航栏右侧，距右边缘16px
尺寸：圆点直径8px，标签文字12px
颜色：
  - 在线+AI：#52C41A（绿色）
  - 在线无AI：#FAAD14（黄色）
  - 离线：无指示器
标签文字：12px / Regular / rgba(0,0,0,0.45)
间距：圆点与文字间距4px
```

#### 交互行为

- **悬停（桌面端）**：显示Tooltip，展示详细信息
  - 在线+AI："已连接，AI智能功能可用"
  - 在线无AI："已连接，AI服务暂时不可用"
  - 离线："当前为离线状态，数据将在联网后自动同步"
- **点击**：展开网络状态详情面板（可选）

### 2.2 状态切换过渡

#### 动效规范

| 过渡类型 | 动画效果 | 时长 | 缓动函数 |
|----------|----------|------|----------|
| 状态指示器颜色变化 | 颜色渐变过渡 | 300ms | ease-in-out |
| 标签文字切换 | 淡入淡出（opacity 0→1） | 300ms | ease-in-out |
| 指示器出现 | 缩放+淡入（scale 0.5→1 + opacity 0→1） | 300ms | ease-in-out |
| 指示器消失（离线） | 淡出（opacity 1→0） | 300ms | ease-in-out |

#### 切换规则

- **不弹窗**：状态切换时仅更新指示器，不弹出任何对话框或通知
- **不中断**：用户正在编辑内容时，状态切换不影响编辑操作
- **防抖处理**：网络状态频繁切换时，取最后一次稳定状态（防抖500ms）

## 三、AI功能按钮交互设计

### 3.1 状态机设计

AI功能按钮遵循以下状态机流转：

```
┌─────────────────────────────────────────────────────────┐
│                    [AI功能按钮]                          │
│                       │                                  │
│                   用户点击                               │
│                       │                                  │
│                       ▼                                  │
│              ┌─────────────────┐                         │
│              │  检测AI可用性    │                         │
│              └────┬────────┬───┘                         │
│                   │        │                             │
│              可用 │        │ 不可用                      │
│                   ▼        ▼                             │
│     ┌──────────────┐  ┌──────────────────────────┐      │
│     │ 显示加载动画  │  │ 显示Toast提示            │      │
│     │ (旋转图标)   │  │ "连接网络后可使用智能功能" │      │
│     └──────┬───────┘  └──────────┬───────────────┘      │
│            │                     │                       │
│            ▼                     │ 2s后自动消失           │
│     ┌──────────────┐             │                       │
│     │  调用AI服务   │             │                       │
│     └──────┬───────┘             │                       │
│            │                     │                       │
│       成功 │   失败              │                       │
│            ▼     ▼               │                       │
│   ┌──────────┐ ┌───────────┐    │                       │
│   │展示结果   │ │错误提示    │    │                       │
│   │(内联/弹窗)│ │Toast提示   │    │                       │
│   └──────────┘ └───────────┘    │                       │
└─────────────────────────────────────────────────────────┘
```

#### 按钮状态定义

| 状态 | 视觉表现 | 交互行为 |
|------|----------|----------|
| 默认 | 紫色描边按钮，✨图标+文字 | 可点击 |
| 加载中 | 按钮内显示旋转加载图标，文字变为"处理中..." | 不可重复点击 |
| 不可用 | 按钮样式不变（不置灰） | 点击后弹出Toast提示 |
| 错误 | 按钮恢复默认状态 | Toast显示错误信息 |
| 成功 | 按钮恢复默认状态 | 展示AI处理结果 |

### 3.2 各模块AI按钮位置

#### 笔记模块

- **按钮名称**：✨ AI摘要
- **位置**：编辑区右上角，与保存按钮同行
- **功能**：对当前笔记内容生成AI摘要
- **结果展示**：在笔记底部插入摘要区块，可编辑/删除
- **离线行为**：点击后Toast提示"连接网络后可使用智能摘要功能"

#### 闪卡模块

- **按钮名称**：✨ AI生成卡片
- **位置**：牌组管理页顶部工具栏，"添加卡片"按钮右侧
- **功能**：根据指定内容自动生成闪卡
- **结果展示**：弹出卡片预览对话框，用户确认后批量添加
- **离线行为**：点击后Toast提示"连接网络后可使用AI生成卡片"

#### 费曼模块

- **按钮名称**：✨ AI评估
- **位置**：步骤2（解释区）底部，提交按钮左侧
- **功能**：AI评估用户对概念的解释是否准确完整
- **结果展示**：在解释区下方显示评估结果卡片（评分+建议）
- **离线行为**：点击后Toast提示"连接网络后可使用AI评估功能"

#### 番茄钟模块

- **按钮名称**：✨ 智能推荐
- **位置**：设置页"专注时长"选项下方
- **功能**：根据历史数据推荐最佳专注时长和休息策略
- **结果展示**：在设置页内联展示推荐方案卡片
- **离线行为**：点击后Toast提示"连接网络后可使用智能推荐功能"

### 3.3 Toast提示规范

```
样式：底部居中，圆角8px，半透明深色背景
背景色：rgba(0, 0, 0, 0.75)
文字：14px / Regular / #FFFFFF
图标：ℹ️ 信息图标（16px）
动画：从底部滑入（translateY 20px→0）+ 淡入，300ms
自动消失：2000ms后自动淡出
堆叠：多个Toast垂直排列，间距8px
```

## 四、离线→在线切换交互

### 4.1 网络恢复流程

```
网络恢复
  │
  ├─ 1. 状态指示器变为绿色（300ms动画过渡）
  │
  ├─ 2. 后台自动检测是否有待同步数据
  │     ├─ 无待同步数据：仅更新指示器，无其他操作
  │     └─ 有待同步数据：自动触发后台同步
  │
  ├─ 3. 同步过程（后台静默执行）
  │     ├─ 显示同步中旋转图标
  │     ├─ 推送本地离线操作日志
  │     ├─ 拉取远端变更
  │     └─ 检测冲突
  │
  ├─ 4. 同步完成
  │     ├─ 无冲突：底部Snackbar提示"数据已同步到云端"（3s自动消失）
  │     └─ 有冲突：弹出冲突解决对话框（见第六节）
  │
  └─ 5. 恢复常态
```

### 4.2 Snackbar提示规范

```
位置：底部居中，距底部24px
尺寸：最大宽度600px，高度40px，圆角8px
背景色：#323232
文字：14px / Regular / #FFFFFF
操作按钮（可选）：右侧，文字#BB86FC
动画：从底部滑入（translateY 100%→0），300ms，ease-out
自动消失：3000ms后滑出消失
```

### 4.3 离线期间用户操作处理

| 操作类型 | 离线行为 | 恢复在线后 |
|----------|----------|------------|
| 创建笔记 | 本地创建并保存 | 自动上传到云端 |
| 编辑笔记 | 本地保存修改记录（操作日志） | 推送操作日志，合并远端变更 |
| 删除笔记 | 本地标记删除（软删除） | 推送删除操作到云端 |
| 创建闪卡 | 本地创建并保存 | 自动上传到云端 |
| 番茄钟记录 | 本地保存记录 | 自动上传到云端 |
| AI功能操作 | Toast提示不可用 | 无（需用户手动重新触发） |

## 五、同步状态指示器

### 5.1 状态定义

| 状态 | 图标 | 动画 | 持续时间 | 说明 |
|------|------|------|----------|------|
| 同步中 | 🔄 旋转箭头 | 顺时针旋转，1s/圈 | 持续至同步结束 | 正在推送/拉取数据 |
| 同步完成 | ✓ 对勾 | 无动画 | 3s后自动消失 | 同步成功完成 |
| 同步失败 | ⚠ 警告三角 | 轻微抖动（shake） | 5s后自动消失 | 同步失败，将自动重试 |

### 5.2 指示器位置

- **桌面端**：与网络状态指示器同行，位于其左侧
- **移动端**：顶部通知区域，临时展示

### 5.3 视觉规范

```
图标尺寸：16px × 16px
颜色：
  - 同步中：#1890FF（蓝色）
  - 同步完成：#52C41A（绿色）
  - 同步失败：#FF4D4F（红色）
旋转动画：linear，1s/圈
抖动动画：translateX(-2px, 2px, -2px, 0)，300ms
过渡动画：淡入淡出，300ms，ease-in-out
```

### 5.4 同步失败重试策略

```
重试次数    等待时间    说明
  1         5s        首次失败后快速重试
  2         15s       第二次重试
  3         30s       第三次重试
  4         60s       第四次重试
  5+        5min      后续重试，最大间隔5分钟
```

重试期间指示器显示"⚠ 同步失败，稍后重试"，用户也可点击手动触发立即重试。

## 六、冲突解决UI设计

### 6.1 冲突对话框设计

#### 对话框结构

```
┌─────────────────────────────────────────────────────┐
│  ⚠ 同步冲突                                    [✕]  │
├─────────────────────────────────────────────────────┤
│                                                     │
│  "检测到以下数据存在冲突，请选择保留版本："           │
│                                                     │
│  ┌──────────────────┬──────────────────────────┐    │
│  │   📱 本地版本     │   ☁️ 云端版本            │    │
│  │   2026-07-08     │   2026-07-08             │    │
│  │   14:30 修改      │   14:25 修改             │    │
│  ├──────────────────┼──────────────────────────┤    │
│  │                  │                          │    │
│  │  [内容预览区域]   │  [内容预览区域]          │    │
│  │                  │                          │    │
│  └──────────────────┴──────────────────────────┘    │
│                                                     │
│  ┌───────────────────────────────────────────────┐  │
│  │  ○ 保留本地版本                                │  │
│  │  ○ 保留云端版本                                │  │
│  │  ○ 两者都保留（生成两个副本）                   │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│                        [取消]    [确认解决]          │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 6.2 批量冲突处理

当存在多个冲突时，使用列表视图：

```
┌─────────────────────────────────────────────────────┐
│  ⚠ 同步冲突（共5项）                           [✕]  │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌───────────────────────────────────────────────┐  │
│  │ ☑ 全选  当前策略：[保留本地版本 ▼]             │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  ┌───────────────────────────────────────────────┐  │
│  │ ☑ 笔记《第三章复习》         [查看详情]       │  │
│  │ ☑ 闪卡组"英语单词"           [查看详情]       │  │
│  │ ☑ 费曼记录"量子力学"         [查看详情]       │  │
│  │ ☐ 笔记《项目计划》           [查看详情]       │  │
│  │ ☐ 番茄钟记录 2026-07-08      [查看详情]       │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│                       [取消]    [批量解决 (3/5)]     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

#### 批量操作规则

- **全选**：一键选中所有冲突项，应用统一策略
- **单选**：逐项选择，可为每项单独设置策略
- **查看详情**：展开单项的本地/云端对比视图
- **批量解决**：仅处理已选中的项目
- **全部解决**：所有项目处理完毕后关闭对话框

### 6.3 冲突预防机制

- **乐观锁**：每条数据携带版本号，写入时检查版本一致性
- **操作日志粒度**：记录字段级变更而非整条记录替换
- **自动合并**：非冲突字段自动合并，仅冲突字段需用户介入
- **冲突窗口期**：同一文档最后编辑后5分钟内不触发同步，等待用户稳定

### 7.2 引导设计原则

| 原则 | 说明 |
|------|------|
| 零门槛 | 全程无需注册/登录/联网即可开始使用 |
| 快速上手 | 每个模块引导不超过3步 |
| 可跳过 | 用户可随时跳过引导，直接进入主界面 |
| 可重看 | 设置中提供"重新查看引导"选项 |
| 轻量 | 引导页不加载网络资源，使用本地动画资源 |

### 7.3 各模块引导步骤

#### 笔记模块引导（3步）

1. **创建**：点击"+"按钮新建笔记
2. **模板**：选择一个笔记模板（高亮展示模板选择器）
3. **编辑**：在编辑区输入内容，体验富文本编辑

#### 闪卡模块引导（3步）

1. **创建牌组**：点击"+"创建你的第一个牌组
2. **添加卡片**：添加一张闪卡（正面问题/背面答案）
3. **开始复习**：点击"开始复习"体验闪卡翻转

#### 费曼模块引导（3步）

1. **选择概念**：输入或选择一个要学习的概念
2. **解释**：用自己的话写下对该概念的理解
3. **回顾**：检查并完善你的解释

#### 番茄钟模块引导（2步）

## 八、模式切换动效规范

### 8.1 动效总则

- **所有过渡动画时长 ≤ 300ms**
- **统一使用 ease-in-out 缓动函数**（cubic-bezier(0.42, 0, 0.58, 1)）
- **不打断用户操作**：动效播放期间用户仍可正常交互
- **尊重系统设置**：检测系统"减少动画"偏好，关闭非必要动画

### 8.2 动效清单

| 场景 | 动画类型 | 时长 | 缓动 | 说明 |
|------|----------|------|------|------|
| 网络状态指示器变色 | 颜色渐变 | 300ms | ease-in-out | 绿↔黄→无 |
| 网络指示器出现/消失 | 缩放+淡入淡出 | 300ms | ease-in-out | scale(0.5→1) + opacity |
| AI按钮加载态 | 图标旋转 | 1s/圈 | linear | 循环至加载完成 |
| Toast弹出 | 上滑+淡入 | 300ms | ease-out | translateY(20px→0) |
| Toast消失 | 下滑+淡出 | 300ms | ease-in | translateY(0→20px) |
| Snackbar弹出 | 上滑+淡入 | 300ms | ease-out | translateY(100%→0) |
| Snackbar消失 | 下滑+淡出 | 300ms | ease-in | translateY(0→100%) |
| 同步图标旋转 | 旋转 | 1s/圈 | linear | 循环至同步完成 |
| 同步完成对勾 | 淡入 | 300ms | ease-in-out | opacity(0→1) |
| 同步失败抖动 | 水平抖动 | 300ms | ease-in-out | translateX(-2→2→-2→0) |
| 冲突对话框出现 | 缩放+淡入 | 300ms | ease-in-out | scale(0.9→1) + opacity |
| 冲突对话框消失 | 缩放+淡出 | 300ms | ease-in-out | scale(1→0.9) + opacity |
| 页面切换 | 左滑/右滑 | 300ms | ease-in-out | translateX |

### 8.3 性能要求

- 所有动画使用 CSS transform/opacity，触发 GPU 加速
- 避免使用 top/left/width/height 等触发布局重排的属性
- 动画帧率目标：60fps
- 低端设备自动降级：减少粒子/模糊效果
