# 课伴数据安全与隐私方案

> 版本：v1.0 | 最后更新：2026-07-08

---

## 一、数据分类与敏感等级

| 数据类型 | 敏感等级 | 存储位置 | 保护措施 | 保留策略 |
|---------|---------|---------|---------|---------|
| **本地笔记/闪卡/学习记录** | 中 | IndexedDB（浏览器本地） | Web Crypto API本地加密；一键导出/删除 | 用户主动删除或注销时清除 |
| **云端同步数据** | 中 | PostgreSQL（服务端） | TLS 1.3传输加密；AES-256服务端加密存储；访问审计 | 注销后30天彻底删除 |
| **AI处理数据** | 中 | 不持久存储 | 明确标注数据去向；仅用于即时响应；不存储原文 | 响应后即丢弃；缓存结果7天 |
| **账户信息（邮箱/密码）** | 高 | PostgreSQL | bcrypt哈希；OAuth2第三方登录；支持注销与数据导出 | 注销后30天彻底删除 |
| **Token/会话信息** | 高 | Redis（服务端）+ localStorage（客户端） | JWT签名验证；短有效期；RefreshToken绑定设备 | AccessToken 15min；RefreshToken 7d |
| **操作日志** | 中 | IndexedDB + PostgreSQL | 本地加密存储；传输加密 | 本地保留7天；服务端保留90天 |
| **统计分析数据** | 低 | 服务端聚合存储 | 匿名化处理；不可关联到个人 | 聚合数据永久保留 |

## 二、本地数据加密方案

### 2.1 加密方案概述

采用 **Web Crypto API**（浏览器原生）对本地敏感数据进行加密，无需引入任何第三方加密库。

**加密层次：**
```
┌─────────────────────────────────────────────────┐
│                  应用层                           │
│  明文数据 ←→ 加密/解密 ←→ StorageAdapter         │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│               Web Crypto API                     │
│  AES-GCM 256位加密 · PBKDF2密钥派生 · RSA签名    │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│              IndexedDB / localStorage             │
│  密文存储（加密后的数据）                          │
└─────────────────────────────────────────────────┘
```

### 2.2 密钥管理方案

```typescript
class KeyManager {
  private static readonly DB_NAME = 'KBanKeys';
  private static readonly KEY_ALGO = 'AES-GCM';
  private static readonly KEY_LENGTH = 256;

  /**
   * 从用户PIN码派生加密密钥（PBKDF2）
   * 用户可选择设置PIN码以增强本地数据保护
   */
  static async deriveKey(pin: string, salt: Uint8Array): Promise<CryptoKey> {
    const encoder = new TextEncoder();
    const keyMaterial = await crypto.subtle.importKey(
      'raw',
      encoder.encode(pin),
      'PBKDF2',
      false,
      ['deriveKey']
    );

    return crypto.subtle.deriveKey(
      {
        name: 'PBKDF2',
        salt,
        iterations: 100000,  // OWASP推荐最低迭代次数
        hash: 'SHA-256',
      },
      keyMaterial,
      { name: this.KEY_ALGO, length: this.KEY_LENGTH },
      false,
      ['encrypt', 'decrypt']
    );
  }

  /**
   * 生成设备级随机密钥（无PIN时使用）
   * 密钥存储在IndexedDB中，设备级别保护
   */
  static async generateDeviceKey(): Promise<CryptoKey> {
    return crypto.subtle.generateKey(
      { name: this.KEY_ALGO, length: this.KEY_LENGTH },
      false,  // 不可导出
      ['encrypt', 'decrypt']
    );
  }

  /**
   * 密钥存储（使用独立的IndexedDB实例）
   */
  static async storeKey(key: CryptoKey): Promise<void> {
    // 密钥以CryptoKey对象存储在IndexedDB中
    // 不可导出（extractable=false），无法被JS代码读取原始密钥
    const db = await this.openKeyDB();
    await db.put('keys', { id: 'device-key', key, createdAt: Date.now() });
  }
}
```

### 2.3 数据加密/解密实现

```typescript
class LocalEncryption {
  private key: CryptoKey;

  constructor(key: CryptoKey) {
    this.key = key;
  }

  /**
   * 加密数据（AES-GCM，自带IV和认证标签）
   */
  async encrypt(plaintext: string): Promise<EncryptedData> {
    const encoder = new TextEncoder();
    const iv = crypto.getRandomValues(new Uint8Array(12)); // 96位随机IV

    const ciphertext = await crypto.subtle.encrypt(
      { name: 'AES-GCM', iv },
      this.key,
      encoder.encode(plaintext)
    );

    return {
      ciphertext: this.arrayBufferToBase64(ciphertext),
      iv: this.arrayBufferToBase64(iv),
      version: 1, // 加密版本号，用于未来算法升级
    };
  }

  /**
   * 解密数据
   */
  async decrypt(encrypted: EncryptedData): Promise<string> {
    const ciphertext = this.base64ToArrayBuffer(encrypted.ciphertext);
    const iv = this.base64ToArrayBuffer(encrypted.iv);

    const plaintext = await crypto.subtle.decrypt(
      { name: 'AES-GCM', iv: new Uint8Array(iv) },
      this.key,
      ciphertext
    );

    return new TextDecoder().decode(plaintext);
  }
}

interface EncryptedData {
  ciphertext: string;  // Base64编码的密文
  iv: string;          // Base64编码的初始化向量
  version: number;     // 加密算法版本
}
```

### 2.4 加密范围

| 数据类型 | 是否加密 | 理由 |
|---------|---------|------|
| 笔记内容（content） | ✅ 加密 | 可能包含个人学习笔记 |
| 闪卡内容（front/back） | ✅ 加密 | 学习内容属用户隐私 |
| 费曼讲解文本 | ✅ 加密 | 个人学习记录 |
| 番茄钟会话记录 | ❌ 不加密 | 仅时间戳+时长，无敏感内容 |
| 操作日志 | ❌ 不加密 | 临时数据，7天后清理 |
| 用户配置 | ❌ 不加密 | 主题、偏好等无敏感信息 |
| 文件夹/标签名 | ❌ 不加密 | 组织信息，低敏感度 |

## 三、传输安全方案

### 3.1 传输层加密

**所有客户端到服务端的通信均使用TLS 1.3：**

```
客户端 ──── TLS 1.3 ────→ Nginx Ingress ────→ 后端服务
         (HTTPS强制)       (TLS终结)         (内网HTTP)
```

**TLS配置要求：**
- 最低TLS版本：TLS 1.2（优先TLS 1.3）
- 禁用弱密码套件：仅允许ECDHE + AES-GCM / ChaCha20-Poly1305
- HSTS：`Strict-Transport-Security: max-age=31536000; includeSubDomains`
- 证书管理：Let's Encrypt自动续签（或云服务托管证书）

### 3.2 API认证机制

### 3.3 请求签名（防篡改）

```typescript
// 对关键API请求添加签名（防止中间人篡改请求体）
async function signRequest(body: string, secret: string): Promise<string> {
  const encoder = new TextEncoder();
  const key = await crypto.subtle.importKey(
    'raw', encoder.encode(secret),
    { name: 'HMAC', hash: 'SHA-256' },
    false, ['sign']
  );

  const signature = await crypto.subtle.sign(
    'HMAC', key, encoder.encode(body)
  );

  return Array.from(new Uint8Array(signature))
    .map(b => b.toString(16).padStart(2, '0')).join('');
}

// 请求头添加签名
// X-Request-Signature: <hmac_signature>
// X-Request-Timestamp: <timestamp>  // 防重放，服务端验证时间差<5分钟
```

### 3.4 数据同步传输加密

```
客户端操作日志（明文）
    ↓
AES-GCM加密（使用会话密钥，每次同步会话生成临时密钥）
    ↓
TLS 1.3传输
    ↓
服务端解密 → 处理 → 加密响应 → TLS返回
```

## 四、合规要求

### 4.1 《个人信息保护法》合规

| 条款 | 要求 | 课伴实现 |
|------|------|---------|
| 第六条（最小必要） | 仅收集必要个人信息 | 仅收集邮箱（注册必需），不收集手机、身份证等 |
| 第七条（公开透明） | 明确告知数据处理方式 | 隐私政策明确列出所有数据收集/处理行为 |
| 第十三条（同意） | 获取用户同意 | 注册时勾选隐私政策；AI功能首次使用时单独授权 |
| 第十四条（撤回同意） | 用户可撤回同意 | 设置页可随时撤回AI数据使用授权 |
| 第二十五条（自动化决策） | AI决策可解释、可申诉 | AI评估结果仅供参考，不影响任何实质权益 |
| 第四十七条（删除权） | 用户有权删除个人数据 | 一键导出+一键删除所有数据 |

### 4.2 《未成年人保护法》合规

课伴目标用户包含大量学生（可能未成年），需额外关注：

| 条款 | 要求 | 课伴实现 |
|------|------|---------|
| 第七十二条（信息安全） | 保护未成年人个人信息 | 本地优先架构，最小化数据上传 |
| 第七十四条（网络沉迷） | 防沉迷机制 | 番茄钟内置休息提醒；连续使用2小时弹窗提示休息 |
| 第七十六条（直播打赏） | 不得提供诱导性消费 | 无打赏功能；AI功能配额制，无付费加速 |

### 4.3 隐私政策模板

```
课伴隐私政策（摘要）

我们收集什么：
- 邮箱地址（注册云端同步时）
- 学习数据（笔记、闪卡等，默认存储在您的设备本地）

我们如何使用：
- 邮箱仅用于登录和多设备同步
- 学习数据仅在您启用同步时上传到服务器
- AI功能使用时，笔记内容会发送到AI服务处理（处理后不保留原文）

我们不会：
- 出售您的任何数据
- 将您的学习数据用于广告投放
- 在未告知的情况下共享数据给第三方

您的权利：
- 随时导出所有个人数据
- 随时删除账户和所有关联数据
- 随时撤回AI功能的数据使用授权
```

### 5.2 数据删除

**一键删除所有数据：**

```typescript
interface DataDeletionService {
  /**
   * 删除本地所有数据
   */
  async deleteLocalData(): Promise<void> {
    // 1. 清空IndexedDB所有表
    await db.transaction('rw',
      [db.notes, db.flashcards, db.decks, db.pomodoroSessions,
       db.feynmanSessions, db.operationLogs, db.folders],
      async () => {
        await db.notes.clear();
        await db.flashcards.clear();
        await db.decks.clear();
        await db.pomodoroSessions.clear();
        await db.feynmanSessions.clear();
        await db.operationLogs.clear();
        await db.folders.clear();
      }
    );

    // 2. 清空localStorage
    localStorage.clear();

    // 3. 删除加密密钥
    await KeyManager.deleteAllKeys();

    // 4. 清除Service Worker缓存
    const cacheNames = await caches.keys();
    await Promise.all(cacheNames.map(name => caches.delete(name)));
  }

  /**
   * 删除云端账户和所有数据
   * 30天冷静期后彻底删除
   */
  async deleteAccount(): Promise<void> {
    await fetch('/api/account/delete', {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${getToken()}` },
    });

    // 本地数据同步清除
    await this.deleteLocalData();

    // 注销Service Worker
    const registration = await navigator.serviceWorker.getRegistration();
    await registration?.unregister();
  }
}
```

### 5.3 账户注销流程

```
用户点击"注销账户"
    ↓
二次确认（输入密码验证身份）
    ↓
显示注销影响说明：
  - 所有云端数据将在30天后永久删除
  - 30天内可登录撤销注销
  - 本地数据不受影响（可继续使用离线功能）
    ↓
用户确认注销
    ↓
服务端标记账户为"待删除"状态
    ↓
返回注销确认邮件
    ↓
30天冷静期
    ├─ 用户登录 → 撤销注销，恢复正常状态
    └─ 30天到期 → 定时任务永久删除所有数据
                  - 用户账户记录
                  - 所有同步数据
                  - 操作日志
                  - AI调用记录
```

### 6.3 安全事件响应流程

```
发现安全事件
    ↓
评估严重程度（P0-P3）
├─ P0（数据泄露）：1小时内响应，立即通知受影响用户
├─ P1（认证绕过）：4小时内响应，临时禁用受影响功能
├─ P2（XSS/CSRF）：24小时内响应，发布修复版本
└─ P3（低危漏洞）：7天内响应，纳入下一个发布周期
    ↓
修复漏洞
    ↓
发布安全公告
    ↓
复盘总结，更新安全规范
```
