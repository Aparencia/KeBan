# 课伴UI设计规范

> 版本：v1.0 | 最后更新：2026-07-08

---

## 一、六大核心体验原则

### 原则1：本地优先，随时可用

> 用户在任何网络环境下均可完整使用所有核心功能，不因网络状态中断学习流程。

**设计体现：**
- 离线状态不显示任何"功能不可用"的禁用按钮或灰色区域
- 状态栏仅以小图标指示网络状态，不弹窗不打断
- AI功能按钮始终可见且可点击，点击后根据可用性给出友好反馈
- 所有数据操作（创建笔记、添加闪卡等）即时响应，无需等待网络

### 原则2：沉浸式学习

> 学习过程中最大限度减少UI元素干扰，让用户专注于内容本身。

**设计体现：**
- 番茄钟运行时：全屏倒计时 + 极简控制按钮，隐藏导航栏和侧边栏
- 笔记编辑时：写作模式（类似iA Writer）隐藏工具栏，仅保留文本区域
- 闪卡复习时：全屏卡片展示，评分按钮仅轻触屏幕底部
- 费曼讲解时：大面积文本编辑区域，概念名称固定在顶部

### 原则3：减少干扰

> 不主动弹窗、不强制引导、不使用红色警告色（除番茄钟品牌色外）。

**设计体现：**
- 通知系统：仅在底部显示轻量Toast，不居中弹窗
- 操作确认：仅删除操作需要确认，创建/编辑即时生效
- 错误提示：温和的橙色/黄色，避免刺眼的红色
- 自动保存：所有编辑内容自动保存，无需手动Ctrl+S

### 原则4：零学习成本

> 新用户无需任何教程即可上手使用，交互符合直觉。

**设计体现：**
- 首次进入每个模块显示3步引导（可跳过），不强制走完
- 图标配文字标签（不依赖纯图标传达含义）
- 操作路径不超过3步（从首页到任何功能）
- 统一的交互模式：长按=更多选项，左滑=删除，下拉=刷新

### 原则5：学习成就感

> 通过数据可视化和微动效强化正反馈，激励持续学习。

**设计体现：**
- 番茄钟完成：卡片翻转动画 + 时长数字滚动增长
- 连续打卡：热力图逐渐填满（类似GitHub贡献图）
- 闪卡掌握：进度条平滑增长 + 卡片堆叠视觉效果
- 费曼评分：雷达图动态绘制 + 评分数字跳动

### 原则6：隐私友好

> 视觉上让用户感知到数据安全，建立信任感。

**设计体现：**
- 设置页清晰展示数据存储位置："所有数据存储在您的设备本地"
- 导出/删除按钮醒目且易找到
- AI功能明确标注"数据仅用于即时处理，不长期存储"
- 不使用"云端""上传"等可能引起隐私顾虑的词汇

## 四、间距规则

### 4.1 基础间距系统

采用 **4px基准** 的间距系统：

```
间距级别：
  xs   4px   紧凑元素间距（图标与文字之间）
  sm   8px   同类元素间距（列表项内部）
  md   16px  区块内间距（卡片内部padding）
  lg   24px  区块间间距（卡片与卡片之间）
  xl   32px  区域间间距（模块标题与内容之间）
  2xl  48px  页面级间距（页面顶部/底部安全区）
```

## 六、图标风格

### 6.1 设计规范

- **风格**：线性图标（Outline），笔画粗细统一
- **笔画宽度**：1.5px（标准）/ 2px（强调）
- **尺寸**：20×20px（导航）/ 24×24px（按钮）/ 16×16px（内联）
- **圆角**：与笔画宽度一致（1.5px圆角）
- **颜色**：跟随文字颜色（currentColor），特殊状态用语义色

## 七、动效规范

### 7.1 动效原则

| 原则 | 说明 |
|------|------|
| **有意义** | 动效传达状态变化或空间关系，不为装饰 |
| **快速** | 微交互≤200ms，过渡≤400ms，不阻塞用户操作 |
| **可关闭** | 用户可在设置中关闭所有动效（`prefers-reduced-motion`） |
| **一致性** | 同类操作使用相同的动效模式 |

### 7.2 核心动效

#### 卡片翻转过渡

```
触发：点击闪卡任意位置
动画：
  - 前半段(0-150ms)：正面绕Y轴旋转0°→90°，scale: 1→0.95
  - 后半段(150-300ms)：背面绕Y轴旋转90°→0°，scale: 0.95→1
缓动函数：ease-in-out
总时长：300ms
3D透视：perspective: 1000px

CSS实现：
  transform-style: preserve-3d;
  transition: transform 300ms ease-in-out;
  &.flipped { transform: rotateY(180deg); }
```

#### 计时器进度动画

```
触发：计时器运行中，每帧更新
动画：
  - SVG环形进度条平滑递减
  - 数字变化使用翻牌动画（每位数字独立翻转）
  - 最后10秒：数字变红，轻微脉动（scale: 1→1.05→1，周期1s）
帧率：requestAnimationFrame (60fps)
数字翻牌：
  - 单数字切换时长：150ms
  - 缓动：ease-out
```

#### 模式切换微动画

```
触发：切换功能模块（番茄钟↔笔记↔闪卡↔费曼）
动画：
  - 旧页面：向左滑出 + 透明度降低（opacity: 1→0）
  - 新页面：从右滑入 + 透明度升高（opacity: 0→1）
  - 底部导航栏图标：弹性放大(scale: 1→1.2→1)
缓动函数：cubic-bezier(0.4, 0, 0.2, 1)（Material标准缓动）
总时长：250ms
```

#### 成就达成动画

```
触发：完成番茄钟、连续打卡、掌握全部闪卡等
动画序列（总计1500ms）：
  1. 0-200ms：成就徽章从底部弹出（translateY: 100%→0，scale: 0→1.2→1）
  2. 200-500ms：徽章周围粒子爆散（8-12个彩色圆点向外扩散）
  3. 500-1000ms：成就文字淡入 + 上移（opacity: 0→1，translateY: 10px→0）
  4. 1000-1500ms：整体停留展示
  5. 1500ms+：自动淡出消失
```

#### 列表项增删动画

```
新增项：
  - 从上方滑入（translateY: -20px→0）
  - 高度从0展开（height: 0→auto，通过max-height模拟）
  - 时长：200ms，ease-out

删除项：
  - 向左滑出（translateX: 0→-100%）
  - 高度收缩到0（height: auto→0）
  - 时长：200ms，ease-in
```

## 八、多端适配策略

### 8.1 端优先级

| 优先级 | 平台 | 技术方案 | 阶段 | 说明 |
|--------|------|---------|------|------|
| **P0** | Web PWA | React + Vite + Service Worker | MVP-1 | 核心平台，优先开发 |
| **P1** | 桌面端 (Win/Mac/Linux) | Electron | MVP-2 | 复用PWA代码，包装为桌面应用 |
| **P2** | 移动端 (iOS/Android) | React Native | V2.0 | 重做原生UI，复用业务逻辑 |

### 8.2 PWA响应式布局（P0/MVP）

**断点设计：**
```
移动端（<768px）：  单栏布局，底部导航
平板端（768-1024px）：双栏布局，侧边导航可折叠
桌面端（>1024px）：  三栏布局，固定侧边导航
```

**移动端布局（<768px）：**
```
┌─────────────────────────┐
│        顶部标题栏         │
├─────────────────────────┤
│                         │
│                         │
│      主内容区域           │
│      (全屏宽度)           │
│                         │
│                         │
│                         │
├─────────────────────────┤
│ 🍅  │ 📝  │ 🃏  │ 🎓  │  底部导航栏
│番茄钟│笔记 │闪卡 │费曼  │
└─────────────────────────┘
```

**桌面端布局（>1024px）：**

### 8.3 Electron桌面端（P1/MVP-2）

```
复用策略：
├─ 完全复用：所有React组件、业务逻辑、存储层
├─ 替换层：StorageAdapter → electron-store（或保持IndexedDB）
├─ 新增能力：
│   ├─ 系统托盘图标 + 番茄钟常驻显示
│   ├─ 全局快捷键（开始/暂停番茄钟）
│   ├─ 原生文件对话框（导入导出）
│   ├─ 系统通知（番茄钟结束提醒）
│   └─ 自动更新（electron-updater）
└─ 打包：electron-builder（Windows MSI/Mac DMG/Linux AppImage）
```

### 8.4 React Native移动端（P2/V2.0）

```
复用策略：
├─ 复用：TypeScript类型定义、SM-2算法、数据模型、API客户端
├─ 重写：全部UI组件（React Native原生组件）
├─ 适配：
│   ├─ 触控手势（左滑删除、下拉刷新）
│   ├─ 原生动画（Reanimated 3）
│   ├─ 本地存储（SQLite → react-native-quick-sqlite）
│   ├─ 推送通知（番茄钟提醒）
│   └─ 应用商店发布（iOS App Store + Google Play）
└─ 共享代码量预估：~30%（业务逻辑+类型+工具函数）
```

### 8.5 触控适配规范

| 元素 | 最小触控区域 | 说明 |
|------|-----------|------|
| 按钮 | 44×44px | Apple HIG推荐最小触控目标 |
| 图标按钮 | 44×44px | 即使图标20px，触控区域扩展到44px |
| 列表项 | 高度≥48px | 手指友好高度 |
| 输入框 | 高度≥44px | 便于手指点击聚焦 |
| 间距 | 按钮间距≥8px | 防止误触 |
