# 课伴本地存储设计文档

> 版本：v1.1 | 最后更新：2026-07-08

> **v1.1变更说明：** 主存储从 Dexie.js (IndexedDB) 升级为 Supabase本地客户端 (PostgreSQL)，统一本地与云端数据栈。

## 一、存储方案对比

| 方案 | 推荐用途 | 优势 | 劣势 | 容量 | 浏览器兼容性 |
|------|---------|------|------|------|------------|
| **Supabase本地客户端 (PostgreSQL)** | **主存储（v1.1）** | 本地云端统一数据栈、完整SQL、RLS策略复用、实时订阅 | Electron优先，PWA纯浏览器场景受限 | 受磁盘限制（理论无限） | Electron完整支持；PWA场景需降级方案 |
| **IndexedDB (via Dexie.js)** | ~~主存储（v1.0，已废弃）~~ / PWA降级方案 | 浏览器原生、大容量、无SQL能力 | API原始、无JOIN、需ORM映射到Supabase | 50MB–2GB | 所有主流浏览器 >98% |
| **SQLite WASM (wa-sqlite)** | 备选/增强 | SQL查询能力强、完整事务支持 | 额外加载~1MB WASM，与Supabase不兼容 | 受OPFS限制 | Chromium系完整 |
| **OPFS** | 大文件存储 | 高性能文件系统API | 仅Chromium完整支持 | 理论无限 | Chromium 102+ |
| **localStorage** | 配置/会话 | 简单同步API、零依赖 | 容量极小、仅字符串 | 5–10MB | 所有浏览器 |

## 二、推荐方案

### 主存储：Supabase本地客户端 (PostgreSQL)

负责存储所有核心业务数据：番茄钟会话记录、笔记内容、闪卡数据、费曼学习记录、操作日志。本地与云端共享同一套PostgreSQL Schema，消除ORM映射开销。

> **v1.1变更：** 原Dexie.js (IndexedDB)方案已废弃。Supabase本地客户端统一本地/云端数据栈，SQL能力完整，RLS策略复用。

### 配置/会话：localStorage

负责存储：用户偏好设置、主题配置、当前选中的牌组ID、计时器模式、窗口尺寸等。

### 大文件：OPFS → IndexedDB Blob降级

- **Chromium浏览器**：笔记中的图片附件、导出的PDF等大文件存储到OPFS
- **其他浏览器**：降级存储到PostgreSQL BYTEA字段或IndexedDB Blob
- 通过 `StorageAdapter` 抽象，上层代码不感知底层差异

### 3.2 Supabase客户端使用示例

```typescript
// lib/supabase-local.ts
import { createClient } from '@supabase/supabase-js';

// 本地Supabase客户端（Electron环境）
const supabase = createClient(
  process.env.SUPABASE_LOCAL_URL,
  process.env.SUPABASE_ANON_KEY
);

// 查询到期闪卡
async function getDueCards(deckId: string) {
  const { data } = await supabase
    .from('flashcards')
    .select('*')
    .eq('deck_id', deckId)
    .lte('due_date', new Date().toISOString())
    .eq('is_suspended', false)
    .order('due_date', { ascending: true });
  return data;
}

// 全文搜索笔记
async function searchNotes(query: string) {
  const { data } = await supabase
    .from('notes')
    .select('id, title, content_text, updated_at')
    .textSearch('content_text', query, { type: 'plain' })
    .order('updated_at', { ascending: false })
    .limit(20);
  return data;
}

// 实时订阅笔记变更
function subscribeNotes(callback: (payload: any) => void) {
  return supabase
    .channel('notes-changes')
    .on('postgres_changes', { event: '*', schema: 'public', table: 'notes' }, callback)
    .subscribe();
}
```

## 四、Schema迁移策略

### 4.1 版本管理原则

Dexie.js内置版本迁移机制，每次Schema变更必须通过版本号递增触发迁移函数。

```typescript
// 版本1：初始Schema
db.version(1).stores({
  notes: 'id, title, folderId, updatedAt',
  flashcards: 'id, deckId, dueDate',
});

// 版本2：新增全文搜索字段 + 笔记标签索引
db.version(2).stores({
  notes: 'id, title, folderId, updatedAt, *tags',
  // flashcards 不变，无需重写
}).upgrade(tx => {
  // 迁移逻辑：为已有笔记生成contentText字段
  return tx.table('notes').toCollection().modify(note => {
    note.contentText = extractPlainText(note.content);
    note.tags = note.tags || [];
  });
});
```

### 4.2 迁移规范

| 规则 | 说明 |
|------|------|
| 版本号严格递增 | 每次Schema变更必须+1，不可跳版本 |
| 迁移函数幂等 | 同一迁移函数执行多次结果相同 |
| 向后兼容 | 新版本必须能读取旧版本数据 |
| 迁移前备份 | 迁移前自动导出当前数据到临时Blob |
| 迁移测试 | 每次迁移必须编写单元测试覆盖 |

### 4.3 迁移失败处理

```
迁移开始 → 备份当前数据
         → 执行upgrade函数
         ├─ 成功：提交事务，删除备份
         └─ 失败：回滚事务，保留原数据
                 → 上报错误日志
                 → 提示用户数据迁移失败，可联系支持
```

### 5.2 容量管理策略

```typescript
class StorageManager {
  // 获取当前存储使用情况
  async getStorageEstimate(): Promise<StorageEstimate> {
    if ('storage' in navigator && 'estimate' in navigator.storage) {
      const { usage, quota } = await navigator.storage.estimate();
      return {
        used: usage ?? 0,
        quota: quota ?? 0,
        percentUsed: ((usage ?? 0) / (quota ?? 1)) * 100,
      };
    }
    return { used: 0, quota: 0, percentUsed: 0 };
  }

  // 容量警告阈值
  async checkStorageHealth(): Promise<StorageHealth> {
    const { percentUsed } = await this.getStorageEstimate();
    if (percentUsed > 90) return { status: 'critical', message: '存储空间即将耗尽' };
    if (percentUsed > 75) return { status: 'warning', message: '存储空间使用较多' };
    return { status: 'healthy', message: '存储空间充足' };
  }

  // 自动清理策略
  async autoClean(): Promise<void> {
    // 1. 清理已同步的操作日志（保留最近7天）
    const cutoff = Date.now() - 7 * 24 * 60 * 60 * 1000;
    await db.operationLogs
      .where('synced').equals(1)
      .and(log => log.timestamp < cutoff)
      .delete();

    // 2. 清理孤立的媒体文件（无引用的附件）
    await this.cleanOrphanMedia();

    // 3. 请求更多配额
    if ('storage' in navigator && 'persist' in navigator.storage) {
      await navigator.storage.persist();
    }
  }
}
```

## 六、索引策略

### 6.1 核心查询场景与索引映射

| 查询场景 | 使用索引 | 说明 |
|---------|---------|------|
| 获取今日番茄钟统计 | `startTime` | `where('startTime').above(todayStart)` |

| 获取某牌组所有闪卡 | `deckId` | `where('deckId').equals(deckId)` |
| 获取未同步操作日志 | `synced` | `where('synced').equals(0)` |
| 按实体查找操作日志 | `entityType + entityId` | 复合查询 |

```typescript
// 复合查询示例：获取某牌组中到期的闪卡
async function getDueCardsByDeck(deckId: string): Promise<Flashcard[]> {
  const now = Date.now();
  return db.flashcards
    .where('deckId').equals(deckId)
    .and(card => card.dueDate <= now && !card.isSuspended)
    .sortBy('dueDate');
}

async function getWeeklySessionsByCourse(courseId: string): Promise<PomodoroSession[]> {

  return db.pomodoroSessions
    .where('courseId').equals(courseId)
    .and(s => s.startTime >= weekAgo)
    .reverse()
    .sortBy('startTime');
}
```

### 6.3 全文搜索实现

IndexedDB原生不支持全文搜索，采用以下方案：

```typescript
// 方案：内存倒排索引 + 持久化到IndexedDB
class FullTextSearch {
  private index: Map<string, Set<string>> = new Map(); // term → noteIds

  // 构建/更新索引
  indexNote(note: Note): void {
    const terms = this.tokenize(note.contentText + ' ' + note.title);
    for (const term of terms) {
      if (!this.index.has(term)) this.index.set(term, new Set());
      this.index.get(term)!.add(note.id!);
    }
  }

  // 搜索
  search(query: string): string[] {
    const terms = this.tokenize(query);
    const resultSets = terms.map(t => this.index.get(t) ?? new Set());
    // 交集：所有词都必须出现
    return [...resultSets.reduce((a, b) => new Set([...a].filter(x => b.has(x))))];
  }

  // 中文分词（简化版：按字符bigram）
  private tokenize(text: string): string[] {
    const normalized = text.toLowerCase().replace(/[^\w\u4e00-\u9fff]/g, ' ');
    const tokens: string[] = [];
    // 英文单词
    tokens.push(...normalized.match(/[a-z]+/g) ?? []);
    // 中文bigram
    const chinese = normalized.match(/[\u4e00-\u9fff]+/g) ?? [];
    for (const segment of chinese) {
      for (let i = 0; i < segment.length - 1; i++) {
        tokens.push(segment.slice(i, i + 2));
      }
      tokens.push(...segment); // 单字也加入
    }
    return [...new Set(tokens)];
  }
}
```

### 7.2 导入格式

**导入策略选项：**

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| `merge` | 合并：ID冲突时保留较新版本 | 多设备数据合并 |
| `replace` | 替换：完全覆盖目标表数据 | 数据恢复/迁移 |
| `skip` | 跳过：ID冲突时保留原有数据 | 增量导入 |
| `duplicate` | 复制：冲突时创建新ID副本 | 从他人导入牌组 |

**导入流程：**
```
用户上传文件
    ↓
格式校验（JSON Schema验证）
    ├─ 格式正确：继续
    └─ 格式错误：提示具体错误位置
    ↓
预览导入内容（展示将要导入的数据摘要）
    ↓
用户选择导入策略
    ↓
执行导入（在事务内完成）
    ├─ 成功：显示导入结果
    └─ 失败：回滚，显示失败原因
```

## 附录：StorageAdapter完整接口

```typescript
interface StorageAdapter {
  // 初始化（应用启动时调用）
  init(): Promise<void>

  // 基础CRUD
  get<T>(store: string, id: string): Promise<T | undefined>
  getAll<T>(store: string, options?: { limit?: number; offset?: number }): Promise<T[]>
  put<T extends { id?: string }>(store: string, item: T): Promise<string>
  delete(store: string, id: string): Promise<void>
  clear(store: string): Promise<void>

  // 批量操作（事务内执行）
  batchPut<T>(store: string, items: T[]): Promise<string[]>
  batchDelete(store: string, ids: string[]): Promise<void>

  // 查询
  where<T>(store: string, field: string, op: Operator, value: any): Promise<T[]>
  count(store: string, field?: string, value?: any): Promise<number>

  // 实时订阅
  subscribe<T>(store: string, id: string, callback: (item: T) => void): () => void
  subscribeQuery<T>(store: string, query: Query, callback: (items: T[]) => void): () => void

  // 导入导出
  exportStores(stores: string[]): Promise<Blob>
  importStores(blob: Blob, strategy: ImportStrategy): Promise<ImportResult>

  // 存储管理
  getStorageEstimate(): Promise<StorageEstimate>
  compact(): Promise<void>  // 清理碎片，压缩存储
}

type Operator = '==' | '!=' | '<' | '<=' | '>' | '>=' | 'in' | 'contains';
type ImportStrategy = 'merge' | 'replace' | 'skip' | 'duplicate';
```
