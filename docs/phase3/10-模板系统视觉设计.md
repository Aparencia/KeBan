> 版本：v1.1 | 最后更新：2026-07-08

## 一、模板系统概述

课伴笔记模块支持多种结构化笔记模板，帮助学生按不同学习场景高效组织内容。模板系统基于 TipTap Extension 体系构建，每个模板对应一个独立的 TipTap Extension，通过 Schema 定义文档结构，通过自定义 NodeView 实现视觉渲染。

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| 降低认知负荷 | 预设结构引导用户聚焦内容，无需从零排版 |
| 场景化覆盖 | 3-5个核心模板覆盖课堂记录、系统学习、头脑风暴等场景 |
| 技术可扩展 | 基于 TipTap Extension，新增模板无需改动核心编辑器代码 |
| 视觉一致性 | 统一的设计语言、配色体系、间距规范 |

### 1.2 核心模板清单

| 编号 | 模板名称 | 适用场景 | 复杂度 |
|------|---------|---------|--------|
| T1 | 大纲式笔记 | 课堂知识要点记录 | ★★☆☆ |
| T2 | 康奈尔笔记法 | 系统性课程学习 | ★★★☆ |
| T3 | 思维导图式 | 头脑风暴、知识关联 | ★★★★ |
| T4 | 自由笔记 | 灵活记录 | ★☆☆☆ |
| T5 | 问答笔记（可选） | 预习/复习 | ★★☆☆ |

## 二、核心模板详细设计

### 2.1 模板1：大纲式笔记

**结构定义：**
```
标题 → 多级缩进列表（最多4级）→ 要点标记（●◆▪▫）
```

**视觉风格：**
- 清晰的层级缩进，左侧竖线连接同级项
- 每级使用不同缩进量（每级+24px）
- 要点标记随层级变化：实心圆 → 菱形 → 实心方块 → 空心圆

**配色规范：**

| 层级 | 文字颜色 | 字号 | 字重 | 竖线颜色 |
|------|---------|------|------|---------|
| 一级标题 | #1F2937（深灰） | 18px | Bold | #3B82F6（蓝色） |
| 二级 | #374151 | 16px | SemiBold | #60A5FA |
| 三级 | #4B5563 | 15px | Medium | #93C5FD |
| 四级 | #6B7280 | 14px | Regular | #BFDBFE |

**间距规范：**
- 同级项间距：8px
- 跨级缩进：每级+24px（左margin）
- 竖线宽度：2px，位于缩进区域内

**TipTap Extension 定义：**
```typescript
const OutlineTemplate = Extension.create({
  name: 'outlineTemplate',

  addKeyboardShortcuts() {
    return {
      Tab: () => this.editor.commands.sinkListItem('listItem'),
      'Shift-Tab': () => this.editor.commands.liftListItem('listItem'),
    };
  },

  addNodeView() {
    return ({ node }) => {
      const dom = document.createElement('div');
      dom.classList.add('outline-level', `outline-level-${node.attrs.level}`);
      return { dom };
    };
  },
});
```

**CSS样式：**
```css
.outline-level { padding-left: calc(var(--level) * 24px); position: relative; }
.outline-level::before {
  content: '';
  position: absolute;
  left: calc(var(--level) * 24px - 8px);
  top: 0; bottom: 0;
  width: 2px;
  background: var(--line-color);
}
```

### 2.2 模板2：康奈尔笔记法

**结构定义：**
```
页面分三区：
┌────────────────────────────────────┐
│         标题/课程信息              │
├───────────┬────────────────────────┤
│  线索栏   │     笔记栏             │
│  (30%)    │     (70%)              │
│           │                        │
│  关键词    │  详细内容              │
│  问题      │  图表                  │
│  提示      │  示例                  │
├───────────┴────────────────────────┤
│         总结栏（100%宽）            │
│         用自己的话总结核心要点       │
└────────────────────────────────────┘
```

**视觉风格：**
- 三区之间有明确的分隔线（虚线，1px dashed）
- 线索栏固定宽度30%，笔记栏自适应70%
- 总结栏固定在底部，可折叠展开

**配色规范：**

| 区域 | 背景色 | 边框颜色 | 文字颜色 | 说明 |
|------|--------|---------|---------|------|
| 标题区 | #FFFFFF | 底部1px solid #E5E7EB | #1F2937 | 课程名+日期+主题 |
| 线索栏 | #EFF6FF（浅蓝） | 右侧1px dashed #93C5FD | #1E40AF | 关键词/提示问题 |
| 笔记栏 | #FFFFFF | 无特殊边框 | #374151 | 主体笔记内容 |
| 总结栏 | #FEF9C3（浅黄） | 顶部1px dashed #FCD34D | #78350F | 用自己的话总结 |

**尺寸规范：**
```
标题区高度：48px（固定）
线索栏宽度：30%（最小200px）
笔记栏宽度：70%（自适应）
总结栏高度：最小100px，最大200px（可拉伸）
分隔线：1px dashed，颜色见配色表
```

**TipTap Extension 定义：**
```typescript
const CornellTemplate = Extension.create({
  name: 'cornellTemplate',

  addGlobalAttributes() {
    return [{
      types: ['paragraph'],
      attributes: {
        cornellZone: { default: null }, // 'cue' | 'notes' | 'summary'
      },
    }];
  },

  addNodeView() {
    return () => {
      const dom = document.createElement('div');
      dom.classList.add('cornell-layout');
      // 三栏布局：grid-template-columns: 30% 70%
      // grid-template-rows: auto 1fr auto
      return { dom };
    };
  },
});
```

### 2.3 模板3：思维导图式

**结构定义：**
```
中心主题
├── 分支主题 A
│   ├── 叶子节点 A1
│   ├── 叶子节点 A2
│   └── 叶子节点 A3
├── 分支主题 B
│   ├── 叶子节点 B1
│   └── 叶子节点 B2
└── 分支主题 C
    └── 叶子节点 C1
```

**视觉风格：**
- 放射状布局，中心节点位于画面中央
- 节点之间用贝塞尔曲线连接
- 不同分支使用不同颜色（自动循环）
- 节点大小随层级递减（中心>分支>叶子）

**配色规范（循环色系）：**

| 分支编号 | 节点背景 | 节点文字 | 连线颜色 |
|---------|---------|---------|---------|
| 1 | #DBEAFE（浅蓝） | #1E40AF | #3B82F6 |
| 2 | #D1FAE5（浅绿） | #065F46 | #10B981 |
| 3 | #FEF3C7（浅黄） | #92400E | #F59E0B |
| 4 | #FCE7F3（浅粉） | #9D174D | #EC4899 |
| 5 | #E0E7FF（浅紫） | #3730A3 | #6366F1 |
| 6 | #FEE2E2（浅红） | #991B1B | #EF4444 |

**节点样式：**
```
中心节点：圆角矩形，padding 16px 24px，背景 #1F2937，文字 #FFFFFF，字号 18px Bold
分支节点：圆角矩形，padding 8px 16px，背景见配色表，字号 14px SemiBold
叶子节点：圆角矩形，padding 6px 12px，背景见配色表，字号 13px Regular
连线：贝塞尔曲线，线宽2px，颜色与所属分支一致
```

**交互行为：**
- 双击节点：进入编辑模式
- 拖拽节点：调整位置（自动重排连线）
- Tab键：添加子节点
- Enter键：添加同级节点
- Delete键：删除节点（含子节点）

## 四、模板渲染技术规范

### 4.1 基于TipTap Extension实现

每个模板对应一个独立的 TipTap Extension，通过以下机制实现：

```typescript
// 模板Extension接口
interface TemplateExtension {
  // Extension名称（唯一标识）
  name: string;

  // 模板Schema定义（节点类型、属性、约束）
  schema: NodeSpec;

  // 自定义NodeView（视觉渲染）
  nodeView: (props: NodeViewProps) => NodeViewComponent;

  // 模板初始内容（新建时的默认结构）
  initialContent: () => JSONContent;

  // 模板元数据
  meta: {
    displayName: string;
    description: string;
    icon: string;
    thumbnail: string;
  };
}
```

### 4.2 模板Schema定义（JSON格式）

```json
{
  "templateId": "cornell",
  "version": "1.0",
  "name": "康奈尔笔记法",
  "schema": {
    "nodes": {
      "cornellDocument": {
        "content": "cornellHeader cornellBody cornellSummary",
        "group": "block",
        "topNode": true
      },
      "cornellHeader": {
        "content": "paragraph",
        "attrs": {
          "courseName": { "default": "" },
          "date": { "default": "" },
          "topic": { "default": "" }
        }
      },
      "cornellBody": {
        "content": "cornellCueColumn cornellNoteColumn",
        "attrs": {
          "cueWidth": { "default": 30 }
        }
      },
      "cornellCueColumn": {
        "content": "block+",
        "attrs": { "zone": { "default": "cue" } }
      },
      "cornellNoteColumn": {
        "content": "block+",
        "attrs": { "zone": { "default": "notes" } }
      },
      "cornellSummary": {
        "content": "block+",
        "attrs": { "zone": { "default": "summary" } }
      }
    }
  }
}
```

### 4.3 模板切换时的内容迁移策略

当用户在已有笔记中切换模板时，需要处理内容迁移：

| 迁移场景 | 策略 | 说明 |
|---------|------|------|
| 任意→自由笔记 | 提取纯文本，放入自由画布 | 零损耗 |
| 任意→大纲式 | 按段落拆分为多级列表 | 可能丢失格式 |
| 任意→康奈尔 | 内容放入笔记栏，线索栏和总结栏留空 | 保留大部分内容 |
| 大纲式→思维导图 | 一级→中心，二级→分支，三四级→叶子 | 结构对应映射 |
| 任意→问答笔记 | 按段落对拆分，奇数段为问题，偶数段为答案 | 需用户手动调整 |

**迁移流程：**
```
用户选择切换模板
    ↓
弹出确认对话框："切换模板可能影响内容排版，是否继续？"
    ↓
[继续] → 保存当前内容快照（用于回退）
    ↓
执行内容迁移算法
    ↓
渲染新模板视图
    ↓
[取消] → 恢复快照
[确认] → 保存新内容，删除快照
```

## 五、模板扩展机制

### 5.1 用户自定义模板（V2.0规划）

V2.0阶段将支持用户基于现有模板创建自定义模板：

- **模板编辑器**：可视化拖拽编辑模板结构
- **模板Schema导出**：用户自定义Schema以JSON格式保存
- **模板预览**：实时预览自定义模板效果
- **模板命名与分类**：用户可为模板命名并归类

### 5.2 模板市场（V2.0规划）

- **模板分享**：用户可将自定义模板分享给其他用户
- **模板评分**：社区评分机制
- **官方推荐**：课伴官方精选模板
- **模板订阅**：按学科/场景订阅模板包

### 5.3 模板注册机制

```typescript
// 模板注册表
class TemplateRegistry {
  private templates: Map<string, TemplateExtension> = new Map();

  register(template: TemplateExtension): void {
    if (this.templates.has(template.name)) {
      throw new Error(`Template "${template.name}" already registered`);
    }
    this.templates.set(template.name, template);
  }

  get(name: string): TemplateExtension | undefined {
    return this.templates.get(name);
  }

  getAll(): TemplateExtension[] {
    return Array.from(this.templates.values());
  }

  // 懒加载：V2.0用户自定义模板按需加载
  async loadRemote(templateId: string): Promise<TemplateExtension> {
    const module = await import(`/templates/${templateId}.js`);
    this.register(module.default);
    return module.default;
  }
}

// 全局模板注册
const registry = new TemplateRegistry();
registry.register(OutlineTemplate);
registry.register(CornellTemplate);
registry.register(MindMapTemplate);
registry.register(FreeNoteTemplate);
registry.register(QATemplate);
```

## 六、无障碍设计规范

### 6.1 键盘导航

- Tab键：在模板选择器卡片间导航
- Enter/Space：选择当前高亮的模板
- Escape：关闭模板选择器
- 模板内部：支持完整的键盘编辑（Tab缩进、Enter换行等）

### 6.2 屏幕阅读器

- 模板卡片：`role="button"` + `aria-label`描述模板名称和用途
- 康奈尔三区：每个区域有 `aria-label`（"线索栏"/"笔记栏"/"总结栏"）
- 思维导图节点：`role="treeitem"` + `aria-expanded`
- 折叠答案：`aria-expanded` + `aria-controls`

### 6.3 颜色对比度

所有文本颜色与背景色的对比度至少满足 WCAG 2.1 AA级标准（4.5:1）：
- 深灰文字(#1F2937)在白底(#FFFFFF)：对比度 14.7:1 ✓
- 蓝色文字(#1E40AF)在浅蓝底(#EFF6FF)：对比度 8.2:1 ✓
- 棕色文字(#78350F)在浅黄底(#FEF9C3)：对比度 7.1:1 ✓
