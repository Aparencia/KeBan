## 二、各工具使用规范

### 2.1 项目管理工具（飞书项目 / Jira）

**用途**：任务分配、进度跟踪、缺陷管理、迭代规划

**使用规范**：
- **工作项类型**：Epic（史诗）→ Story（用户故事）→ Task（任务）→ Sub-task（子任务）→ Bug（缺陷）
- **状态流转**：待处理 → 进行中 → 待Review → 已完成 → 已关闭
- **优先级定义**：P0（紧急-阻塞性）→ P1（高-本迭代必须）→ P2（中-尽量完成）→ P3（低-可推迟）
- **每日更新**：开发人员每日下班前更新任务状态和进度
- **迭代周期**：2周一个Sprint，Sprint开始日规划，结束日评审

**配置清单**：
- [ ] 创建项目空间
- [ ] 配置工作流（状态机）
- [ ] 设置看板视图（按迭代/按人员/按状态）
- [ ] 配置通知规则（任务分配/状态变更/到期提醒）
- [ ] 导入需求列表（从PRD拆解）
- [ ] 配置燃尽图/燃起图

### 2.2 代码仓库（GitHub / GitLab）

**用途**：源代码版本管理、代码Review、Issue跟踪

**使用规范**：
- **分支策略**：Git Flow（main / develop / feature / release / hotfix）
- **PR/MR规则**：必须至少1人Review通过 + CI检查通过方可合并
- **Commit规范**：遵循 Conventional Commits（`feat: / fix: / docs: / refactor: / test: / chore:`）
- **Tag规范**：遵循 SemVer（`v1.0.0`），每次release打Tag
- **保护分支**：main和develop分支禁止直接push，必须通过PR/MR

**仓库规划**：

| 仓库名 | 内容 | 可见性 |
|--------|------|--------|
| keban-client | 客户端（桌面端+移动端）代码 | 私有 |
| keban-server | 后端服务代码 | 私有 |
| keban-sync-engine | 离线同步引擎 | 私有 |
| keban-ai-service | AI服务模块 | 私有 |
| keban-docs | 项目文档 | 私有 |
| keban-design | 设计资源（Figma导出） | 私有 |

**配置清单**：
- [ ] 创建仓库组/组织
- [ ] 配置分支保护规则
- [ ] 配置CI/CD Pipeline
- [ ] 配置代码扫描（SonarQube/Snyk）
- [ ] 配置PR模板
- [ ] 配置Issue模板

### 2.3 文档协作（飞书文档 / 语雀）

**用途**：项目文档协作编辑、知识沉淀、会议纪要

**使用规范**：
- **目录结构**：按阶段组织（Phase0 / Phase1 / Phase2 / Phase3 / 运维文档）
- **命名规范**：`[阶段]-[序号]-[文档名称]`，如 `P0-007-团队通讯录`
- **版本管理**：重要文档使用内置版本历史功能，重大修改前创建快照
- **评论规范**：使用@提及相关人员，评论需在48h内回复
- **归档规则**：每个阶段结束后归档已完成文档，设置只读权限

**配置清单**：
- [ ] 创建项目知识库空间
- [ ] 建立目录结构
- [ ] 配置权限（按角色设置查看/编辑权限）
- [ ] 迁移现有文档
- [ ] 配置文档模板

### 2.4 设计协作（Figma）

**用途**：UI设计、交互原型、设计规范、开发交付

**使用规范**：
- **项目结构**：按模块组织页面（首页/课程页/笔记页/闪卡页/设置页...）
- **设计规范**：建立统一的设计系统（颜色/字体/间距/组件库）
- **交付规范**：设计稿标注完整，支持开发直接查看CSS属性
- **版本管理**：每次评审通过后创建版本快照
- **开发交付**：使用Figma Dev Mode，开发可直接查看切图和参数

**配置清单**：
- [ ] 创建Figma团队空间
- [ ] 建立设计系统（Design System）
- [ ] 配置组件库
- [ ] 邀请团队成员
- [ ] 配置开发交付模式

### 2.5 API管理（Apifox）

**用途**：API文档管理、接口调试、Mock服务、自动化测试

**使用规范**：
- **API设计**：RESTful风格，遵循OpenAPI 3.0规范
- **文档要求**：每个接口必须包含完整的请求/响应参数说明、示例数据
- **Mock服务**：前端开发阶段使用Apifox Mock服务，解耦前后端开发
- **测试集成**：API自动化测试用例集成到CI/CD流水线

**配置清单**：
- [ ] 创建Apifox项目空间
- [ ] 配置环境（开发/测试/预发/生产）
- [ ] 建立API目录结构
- [ ] 配置Mock规则
- [ ] 配置CI/CD集成

## 三、权限管理策略

| 工具 | PM | 产品经理 | 技术负责人 | 开发工程师 | UI设计师 | 测试工程师 | 客户方 |

| 代码仓库 | 查看 | 查看 | 管理员 | 开发权限 | 查看 | 查看 | - |

| 设计协作 | 查看 | 查看 | 查看 | 查看 | 管理员 | 查看 | 查看 |
| API管理 | 查看 | 查看 | 管理员 | 编辑 | - | 编辑 | - |

| 监控告警 | 查看 | - | 管理员 | 查看 | - | 查看 | - |

### 3.2 权限管理原则

- **最小权限原则**：仅授予完成工作所需的最小权限集
- **角色绑定**：权限与岗位角色绑定，不绑定到具体个人
- **入职配置**：新成员入职当天，由PM按角色模板配置所有工具权限
- **离职清理**：成员离开项目当天，收回所有工具权限
- **定期审计**：每月1日审查各工具权限列表，清理无效授权
- **临时提权**：因工作需要临时提升权限的，需PM审批，72h后自动回收

## 四、数据备份策略

### 4.1 备份计划

| 数据类型 | 备份频率 | 备份方式 | 保留周期 | 存储位置 | 负责人 |
|----------|---------|---------|---------|---------|--------|
| 代码仓库 | 实时（Git推送即同步） | Git远程仓库 + 每日快照 | 永久 | 云端 + 本地镜像 | DevOps |
| 项目文档 | 每日增量 | 平台自动备份 + 每周手动导出 | 1年 | 云端 + NAS | PM |
| 设计文件 | 每次评审后 | Figma版本快照 + 导出备份 | 项目周期内 | 云端 + NAS | UI设计师 |
| API文档 | 每周 | Apifox导出 | 项目周期内 | 代码仓库 | 技术负责人 |
| 数据库 | 每日全量 + 实时binlog | 自动备份脚本 | 30天 | 对象存储（异地） | DevOps |
| 配置文件 | 每次变更 | Git版本管理 | 永久 | 代码仓库 | DevOps |

### 4.2 灾难恢复计划

- **RTO（恢复时间目标）**：核心服务4小时内恢复
- **RPO（恢复点目标）**：数据丢失不超过1小时
- **恢复演练**：每季度进行一次灾难恢复演练，记录演练结果
- **备份验证**：每月验证一次备份数据的完整性和可恢复性
