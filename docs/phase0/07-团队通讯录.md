## 三、紧急联系人

> **说明**：用于生产事故、安全事件等紧急场景的快速联络。

| 场景 | 联系人 | 角色 | 手机（24h） | 备用联系方式 | 响应时效 |
|------|--------|------|-------------|-------------|---------|
| 生产环境事故 | 待定 | DevOps/SRE | - | - | 15分钟内响应 |
| 数据安全事故 | 待定 | 技术负责人 | - | - | 30分钟内响应 |
| 客户方紧急需求 | 待定 | PM | - | - | 2小时内响应 |
| 法律合规问题 | 待定 | 客户方项目负责人 | - | - | 4小时内响应 |
| 服务器宕机/网络故障 | 待定 | 云服务商技术支持 | - | - | 按SLA响应 |

### 紧急联络流程

```
发现问题 → 判断紧急等级 → 通知第一联系人
                              ↓ 5分钟无响应
                         通知第二联系人（备用）
                              ↓ 15分钟无响应
                         升级至PM/技术负责人
                              ↓
                         启动应急响应会议
```

## 四、汇报关系图

```
项目总监
  └── 项目经理（PM）
        ├── 产品经理
        │     ├── UI设计师
        │     └── （需求对接）
        ├── 技术负责人/架构师
        │     ├── 客户端架构师
        │     │     └── 前端工程师（×3-4）
        │     ├── AI算法工程师（×2）
        │     ├── 离线同步专家
        │     ├── 后端工程师（×2-3）
        │     └── DevOps/SRE
        └── 测试工程师（×1-2）
```

### 汇报关系说明

- **实线汇报**（行政管理关系）：决定绩效考核、薪资调整
- **虚线汇报**（业务协作关系）：日常任务分配、技术方向指导
- **跨组协作**：前端工程师与UI设计师之间为虚线协作关系，日常需求通过产品经理协调
- **紧急事项**：可越级汇报，但事后需在24小时内补报直属上级
