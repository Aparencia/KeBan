## 一、项目愿景

> **"提高网课效率"——让每一位网课学生都拥有高效学习的工具。**

课伴不仅仅是一款软件产品，更是对"如何让在线学习更高效"这一命题的深度回答。我们相信：

- 每一个上网课的学生都值得拥有科学的学习工具，而不是在低效中挣扎
- 学习的四大环节——专注、记录、记忆、理解——不应被割裂在多个互不关联的应用中
- 好的工具应当"随时可用"，不应因为网络问题而让学生中断学习
- AI应当是学习的"加速器"，而非"必需品"——核心功能永远不应依赖网络

课伴的愿景是用技术赋能学习，让每一分钟的网课学习都高效有价值。

## 二、项目目标

### 2.1 核心目标

打造**本地优先的智能学习工具**，整合四大核心模块，形成完整的学习闭环：

| 模块 | 核心功能 | 学习方法论 | 解决痛点 |
|------|----------|-----------|----------|
| **番茄钟** | 科学计时、专注统计、休息提醒 | 番茄工作法 | 注意力易分散 |
| **笔记** | 结构化笔记、模板系统、AI摘要 | 康奈尔笔记法/大纲笔记法 | 笔记效率低 |
| **闪卡** | SM-2间隔重复、智能制卡、学习统计 | 间隔重复学习法 | 课后复习无体系 |
| **费曼** | AI学伴、讲解录音、理解评估 | 费曼学习法 | 学习效果难量化 |

### 2.2 技术目标

- **本地优先**：所有核心功能可在纯离线环境下完整运行，打开即用
- **AI增强可选**：网络可用时可选开启AI能力（笔记摘要、智能闪卡、费曼评估）
- **双模式架构**：功能级按需激活，非全局二元切换
- **数据安全**：用户数据完全归属用户，本地存储为主，云同步为辅
- **跨平台**：基于Web技术，支持Chrome/Firefox/Safari/Edge主流浏览器

### 2.3 商业目标

- **快速获客**：免费基础版降低用户决策门槛，目标MVP-1上线后3个月内获取1万注册用户
- **付费转化**：AI增强订阅制，目标首年付费转化率5-8%
- **用户留存**：四大模块形成学习闭环，目标月活跃率>40%

## 三、项目范围

### 3.1 MVP-1范围（本地核心版）

MVP-1聚焦于四大模块的**本地核心功能**完整可用：

**包含（In Scope）：**

| 模块 | 功能范围 |
|------|----------|
| **番茄钟** | 自定义时长、短/长休息、专注统计（日/周/月）、学习时段热力图 |
| **笔记** | 富文本编辑、Markdown支持、文件夹/标签组织、搜索、模板系统、导入/导出 |
| **闪卡** | 手动制卡、SM-2复习调度、学习统计、卡片分组/标签、导入/导出 |
| **费曼** | 文字讲解输入、录音讲解、本地录音回放、讲解时长统计 |
| **通用** | 本地IndexedDB存储、PWA离线安装、数据导入导出、响应式布局 |

**不包含（Out of Scope）：**

- AI笔记摘要、AI智能闪卡生成、AI费曼评估
- 云同步、多设备数据同步
- 用户账户系统、登录注册
- 社交功能（分享、排行榜等）
- 移动端原生App

### 3.2 MVP-2范围（AI增强版）

MVP-2在MVP-1基础上叠加**云端增强功能**：

| 功能类别 | 具体内容 |
|----------|----------|
| **AI笔记摘要** | 一键生成结构化摘要、重点提取、思维导图生成 |
| **智能闪卡生成** | AI从笔记内容自动提取知识点生成闪卡对 |
| **费曼AI评估** | AI扮演学生角色，评估讲解质量，给出理解度评分和改进建议 |
| **云同步** | 基于CRDT的多设备数据同步、冲突自动解决 |
| **用户账户** | 注册/登录、学习数据云端备份、订阅管理 |
| **AI网关** | 多模型路由、Token成本控制、请求缓存、限流保护 |

### 3.3 正式上线范围

在MVP-2基础上完善运营体系和用户体验：

- 完整的用户引导和新手教程
- 运营后台和数据分析平台
- 用户反馈收集和处理系统
- 完善的多语言支持（中文/英文）
- 性能优化和安全加固

## 五、产品哲学

### 5.1 核心理念

> **"核心功能随时可用，AI智能锦上添花"**

这不是一个口号，而是贯穿产品设计、架构、开发的根本原则。每一个功能决策、每一次架构选择，都必须回答一个问题：**这是否违背了"核心功能随时可用"的承诺？**

### 5.2 功能级按需激活

双模式是**功能级按需激活**，而非全局二元切换：

| 设计原则 | 说明 |
|----------|------|
| **独立判断** | 每个功能模块独立判断是否需要网络/AI |
| **优雅降级** | AI功能离线时灰显，不报错，不阻塞任何操作 |
| **无感切换** | 网络恢复后平滑过渡，无需用户手动切换模式 |
| **数据一致** | 离线和在线使用同一份本地数据，不存在"同步"的概念差异 |
| **用户可控** | 用户可选择永远不使用AI功能，核心体验不受影响 |

### 5.3 产品决策优先级

当面临功能取舍时，按以下优先级决策：

```
1. 核心功能离线可用性（最高优先级，不可妥协）
2. 用户体验流畅度
3. 功能完整度
4. AI增强能力
5. 附加功能（最低优先级）
```
