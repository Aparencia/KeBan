## 一、候选方案概述

课伴笔记模块需要一款支持富文本编辑、模板渲染、离线使用、未来可协作的编辑器。以下是四个候选方案的定位分析：

| 方案 | 定位 | 技术栈 | 首次发布 |
|------|------|--------|----------|
| **ProseMirror** | 底层编辑器框架，提供完整文档模型和编辑原语 | 纯JS/TS，框架无关 | 2016 |
| **TipTap** | 基于 ProseMirror 的高层封装框架，开箱即用 | TS，官方 React/Vue 支持 | 2019 |
| **Slate** | React 原生编辑器框架，受 Draft.js 启发 | React + TS | 2016 |
| **Quill** | 轻量级富文本编辑器，Delta 操作模型 | 纯JS，框架无关 | 2014 |

## 三、模板渲染方案验证

### 3.1 康奈尔笔记模板实现测试

康奈尔笔记法将页面分为线索栏（Cue）、笔记栏（Note-taking）、总结栏（Summary）三个区域。

**TipTap 实现方案**：

```typescript
// 康奈尔模板 Extension
const CornellTemplate = Node.create({
  name: 'cornellTemplate',
  group: 'block',
  content: 'cornellCue cornellNote cornellSummary',
  atom: false,
  
  parseHTML() {
    return [{ tag: 'div[data-type="cornell"]' }]
  },
  
  renderHTML({ HTMLAttributes }) {
    return ['div', 
      mergeAttributes(HTMLAttributes, { 'data-type': 'cornell' }),
      0
    ]
  }
})

// 线索栏子节点
const CornellCue = Node.create({
  name: 'cornellCue',
  group: 'block',
  content: 'paragraph+',
  // ...
})
```

**测试结果**：

| 测试项 | ProseMirror | TipTap | Slate | Quill |
|--------|------------|--------|-------|-------|
| 模板结构渲染 | ✅ | ✅ | ✅ | ⚠️ 布局受限 |
| 区域间拖拽限制 | ✅ | ✅ | ✅ | ❌ 无法限制 |
| 模板切换（不丢失内容） | ✅ | ✅ | ⚠️ 需手动迁移 | ❌ 需重建 |
| 导出为 PDF | ✅ | ✅ | ✅ | ⚠️ 格式丢失 |
| 开发耗时 | 3天 | 1天 | 2天 | 4天 |

## 四、性能测试

### 4.1 测试环境

- 浏览器：Chrome 120 / Firefox 121 / Safari 17
- 硬件：模拟中端设备（4核 CPU、8GB RAM）
- 测试数据：10000字中文文本 + 100张内联图片（每张约200KB）

### 4.2 大文档编辑性能

| 操作 | ProseMirror | TipTap | Slate | Quill |
|------|------------|--------|-------|-------|
| 文档加载时间（10000字+100图） | ~120ms | ~180ms | ~350ms | ~90ms |
| 首屏渲染时间 | ~80ms | ~150ms | ~280ms | ~60ms |
| 连续输入延迟（单字符） | ~2ms | ~3ms | ~8ms | ~2ms |
| 粘贴大段文本（5000字） | ~50ms | ~80ms | ~200ms | ~40ms |
| 滚动流畅度（60fps） | ✅ 稳定60fps | ✅ 稳定60fps | ⚠️ 偶尔掉帧(45-55fps) | ✅ 稳定60fps |
| 图片插入延迟 | ~30ms | ~45ms | ~120ms | ~25ms |

### 4.3 输入延迟详细测试

在不同文档规模下的单字符输入延迟（从 keydown 到 DOM 更新完成）：

| 文档规模 | ProseMirror | TipTap | Slate | Quill |
|----------|------------|--------|-------|-------|
| 100字 | ~1.5ms | ~2ms | ~4ms | ~1.5ms |
| 1,000字 | ~2ms | ~3ms | ~6ms | ~2ms |
| 10,000字 | ~2ms | ~3ms | ~8ms | ~2ms |
| 50,000字 | ~3ms | ~4ms | ~15ms | ~3ms |
| 100,000字 | ~4ms | ~5ms | ~30ms（卡顿明显） | ~4ms |

### 4.4 内存占用测试

| 文档规模 | ProseMirror | TipTap | Slate | Quill |
|----------|------------|--------|-------|-------|
| 空文档 | ~3MB | ~5MB | ~4MB | ~2MB |
| 1,000字 | ~5MB | ~8MB | ~7MB | ~3MB |
| 10,000字+100图 | ~25MB | ~35MB | ~60MB | ~18MB |
| 50,000字+500图 | ~80MB | ~110MB | ~180MB（接近上限） | ~55MB |

### 4.5 性能评估结论

- **ProseMirror / Quill**：性能最优，底层引擎效率高
- **TipTap**：在 ProseMirror 基础上增加了约 30-50% 的性能开销，但在可接受范围内
- **Slate**：性能是主要短板，大文档场景下输入延迟和内存占用明显偏高
- 对于课伴的预期文档规模（单篇笔记 <5000字），所有方案性能均可接受

## 五、选型结论

### 5.1 推荐方案：TipTap（基于 ProseMirror）

| 决策因素 | 评分 | 说明 |
|----------|------|------|
| React 原生集成 | ⭐⭐⭐⭐⭐ | 官方 @tiptap/react 包，零配置集成 |
| 开发效率 | ⭐⭐⭐⭐⭐ | Extension 系统开发效率高，预估比 ProseMirror 节省 60% 时间 |
| 模板渲染能力 | ⭐⭐⭐⭐⭐ | 自定义 Node Extension 天然适合模板系统 |
| ProseMirror 底层能力 | ⭐⭐⭐⭐⭐ | 继承 ProseMirror 强大的文档模型和编辑能力 |
| 协作编辑就绪 | ⭐⭐⭐⭐ | Hocuspocus 官方协作方案，未来可直接集成 |
| 社区与生态 | ⭐⭐⭐⭐⭐ | 25k+ star，活跃维护，丰富的社区扩展 |
| 学习曲线 | ⭐⭐⭐⭐ | 3-5天上手，比 ProseMirror 低很多 |
| 性能表现 | ⭐⭐⭐⭐ | 比原生 ProseMirror 有约30%开销，课伴场景可接受 |
| 包大小 | ⭐⭐⭐ | ~200KB，相比其他方案偏大但可接受 |

### 5.2 选择 TipTap 而非其他方案的核心理由

**vs ProseMirror**：
- ProseMirror 底层能力虽强，但课伴 MVP 阶段不需要如此底层的控制
- TipTap 封装了 ProseMirror 90% 的能力，同时开发效率提升 2-3 倍
- 如果后期需要底层控制，可直接访问 ProseMirror 实例（TipTap 不封闭底层）

**vs Slate**：
- Slate 在大文档场景性能较差（输入延迟 8-30ms vs TipTap 3-5ms）
- Slate 项目维护活跃度下降，核心贡献者减少
- Slate 的协作编辑方案尚不成熟

**vs Quill**：
- Quill 的 Delta 格式限制了自定义模板的实现（无法定义结构化节点）
- Quill 不适合课伴的模板渲染需求
- Quill v2 开发进展缓慢，API 可能不稳定

### 5.4 未来协作编辑路径（V2.0）

TipTap 与 Yjs 的集成方案已相当成熟：

```typescript
// V2.0 协作编辑集成示例
import { HocuspocusProvider } from '@hocuspocus/provider'
import { Collaboration } from '@tiptap/extension-collaboration'
import { CollaborationCursor } from '@tiptap/extension-collaboration-cursor'

const provider = new HocuspocusProvider({
  url: 'wss://sync.keban.app',
  name: `note-${noteId}`,
})

const editor = useEditor({
  extensions: [
    StarterKit.configure({ history: false }), // 禁用本地历史，使用CRDT历史
    Collaboration.configure({ document: provider.document }),
    CollaborationCursor.configure({ provider }),
  ],
})
```

### 5.5 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| TipTap Pro 版功能收费 | 部分高级扩展需付费 | 核心功能全部 MIT 开源，仅协作UI组件收费 |
| ProseMirror 大版本更新 | 底层 API 可能变化 | TipTap 团队会适配更新，无需直接处理 |
| 包大小 200KB 偏大 | 首屏加载略慢 | Tree-shaking + 按需加载扩展，预计可压缩到 120KB |
| 移动端编辑体验 | 移动端富文本编辑普遍挑战 | TipTap 移动端基本可用，复杂操作可降级为纯文本 |
