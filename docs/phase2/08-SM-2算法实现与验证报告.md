## 一、SM-2 算法原理

### 1.1 算法起源

SM-2（SuperMemo 2）算法由 Piotr Woźniak 于 1987 年开发，是间隔重复系统（Spaced Repetition System, SRS）的奠基算法之一。该算法通过分析用户对知识的回忆质量，动态调整复习间隔，从而在遗忘发生前安排复习，最大化学习效率。

Anki、SuperMemo、Quizlet 等主流学习工具均采用 SM-2 或其变体作为核心调度引擎。

### 1.2 核心参数

| 参数 | 符号 | 说明 | 初始值 |
|------|------|------|--------|
| 难度因子 | EF (Ease Factor) | 反映知识点难度，越大表示越容易，间隔增长越快 | 2.5 |
| 复习间隔 | I (Interval) | 当前复习间隔（天数） | — |
| 重复次数 | n (Repetitions) | 连续正确回忆的次数 | 0 |
| 回忆质量 | q (Quality) | 用户对回忆质量的评分（0-5分制） | — |
| 到期日期 | dueDate | 下次应复习的日期 | — |

### 1.3 计算公式

**EF 更新公式**：

$$EF' = EF + (0.1 - (5 - q) \times (0.08 + (5 - q) \times 0.02))$$

- 当 q = 5（Easy）：EF' = EF + 0.1（难度因子增大，间隔增长加快）
- 当 q = 4（Good）：EF' = EF - 0.14（难度因子略降）
- 当 q = 3（Hard）：EF' = EF - 0.28（难度因子降低）
- 当 q = 0-1（Again）：EF' = EF - 0.8（难度因子大幅降低）
- **EF 下限**：任何情况下 EF 不低于 1.3

**间隔计算规则**：

| 重复次数 n | 间隔 I(n) |
|-----------|----------|
| n = 1 | 1 天 |
| n = 2 | 6 天 |
| n ≥ 3 | I(n-1) × EF |

**重置条件**：当 q < 3 时，n 重置为 0，重新开始学习序列。

### 1.4 评分映射

Anki 使用的四按钮评分制（Again / Hard / Good / Easy）映射到 SM-2 原始 0-5 分制：

| Anki 按钮 | 对应 SM-2 分值 | 含义 |
|-----------|---------------|------|
| Again | 0-1（取1） | 完全忘记 |
| Hard | 2-3（取3） | 勉强记得，回忆困难 |
| Good | 4 | 记得，有些犹豫 |
| Easy | 5 | 轻松记得 |

## 二、实现代码

### 2.1 核心算法实现

```typescript
/** 卡片调度状态 */
interface CardState {
  easeFactor: number;   // 难度因子，初始 2.5
  interval: number;     // 复习间隔（天）
  repetitions: number;  // 连续正确次数
  dueDate: Date;        // 到期日期
  lapses: number;       // 遗忘次数统计
}

/** 评分枚举 */
enum Rating {
  Again = 0,  // 完全忘记
  Hard  = 1,  // 勉强记得
  Good  = 2,  // 记得
  Easy  = 3,  // 轻松记得
}

/** SM-2 调度函数 */
function sm2(card: CardState, rating: Rating): CardState {
  // 映射四评分到 SM-2 的 0-5 分制
  const qualityMap: Record<Rating, number> = {
    [Rating.Again]: 1,
    [Rating.Hard]:  3,
    [Rating.Good]:  4,
    [Rating.Easy]:  5,
  };
  const q = qualityMap[rating];

  // 计算新的 EF
  let newEF = card.easeFactor + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02));
  newEF = Math.max(newEF, 1.3); // EF 下限保护

  let newInterval: number;
  let newReps: number;
  let newLapses = card.lapses;

  if (rating === Rating.Again) {
    // 忘记：重置学习序列
    newReps = 0;
    newInterval = 1; // 1天后复习（简化版）
    newLapses += 1;
  } else if (card.repetitions === 0) {
    // 首次正确回答
    newReps = 1;
    newInterval = 1;
  } else if (card.repetitions === 1) {
    // 第二次正确回答
    newReps = 2;
    newInterval = 6;
  } else {
    // 后续复习：间隔 × EF
    newReps = card.repetitions + 1;
    newInterval = Math.round(card.interval * newEF);
  }

  // Hard 调整：间隔缩短 40%
  if (rating === Rating.Hard) {
    newInterval = Math.max(1, Math.round(newInterval * 0.6));
  }

  // Easy 调整：间隔增长 30%
  if (rating === Rating.Easy) {
    newInterval = Math.round(newInterval * 1.3);
  }

  // 间隔上限保护（不超过 5 年）
  newInterval = Math.min(newInterval, 1825);

  return {
    easeFactor: newEF,
    interval: newInterval,
    repetitions: newReps,
    dueDate: addDays(new Date(), newInterval),
    lapses: newLapses,
  };
}

/** 日期辅助函数 */
function addDays(date: Date, days: number): Date {
  const result = new Date(date);
  result.setDate(result.getDate() + days);
  return result;
}
```

## 四、四评分制映射方案

### 4.1 Anki 四按钮 vs SM-2 六分制

| SM-2 原始分 | 含义 | Anki 按钮 | 课伴映射 |
|------------|------|----------|----------|
| 0 | 完全不记得 | Again | Again |
| 1 | 错误但看到答案后有印象 | Again | Again |
| 2 | 错误但回忆出部分信息 | Hard（部分） | Hard |
| 3 | 正确但回忆困难 | Hard（主要） | Hard |
| 4 | 正确，有些犹豫 | Good | Good |
| 5 | 正确，轻松记得 | Easy | Easy |

### 4.2 映射策略选择

课伴采用**四按钮映射方案**（与 Anki 一致），将 SM-2 的 0-5 分简化为：

| 按钮 | 映射到 SM-2 分值 | 说明 |
|------|-----------------|------|
| Again | 1 | 取 0-1 区间中间值，表示"遗忘" |
| Hard | 3 | 取 2-3 区间上限，避免过度惩罚 |
| Good | 4 | 标准正确回答 |
| Easy | 5 | 轻松正确回答 |

### 4.3 差异分析

| 差异点 | 标准 SM-2（六分制） | Anki 变体（四按钮） | 课伴实现 |
|--------|---------------------|--------------------|----|
| 评分粒度 | 6级 | 4级 | 4级（与Anki一致） |
| EF 下限 | 1.3 | 1.3 | 1.3 |
| Again 后间隔 | 立即重学 | 1天（或分钟级新卡） | 1天（简化版） |
| Hard 间隔调整 | 无额外调整 | 间隔×1.2（Anki特有） | 间隔×0.6（保守策略） |
| Easy 间隔调整 | 无额外调整 | Easy Bonus 额外加成 | 间隔×1.3 |
| 学习阶段（新卡） | 无区分 | 分学习/复习阶段 | 简化处理，统一调度 |

### 4.4 设计决策说明

- **采用四按钮**：降低用户决策成本，Anki 已验证此交互模式用户接受度高
- **Hard 间隔缩短**：采用 ×0.6 而非 Anki 的 ×1.2，因为课伴面向 K12 学生，知识点关联性更强，Hard 意味着需要更频繁复习
- **简化新卡阶段**：MVP 不区分学习/复习阶段，统一用 SM-2 调度，降低实现复杂度

### 5.3 长期调度模拟（365天学习曲线）

模拟场景：100张新卡，用户每天学习并复习所有到期卡片，评分随机分布（Good 60%、Hard 20%、Easy 15%、Again 5%）。

| 时间节点 | 活跃卡片数 | 平均EF | 平均间隔（天） | 每日复习量 |
|----------|-----------|--------|---------------|-----------|
| 第7天 | 100 | 2.42 | 2.8 | 45 |
| 第30天 | 100 | 2.38 | 12.5 | 18 |
| 第60天 | 100 | 2.35 | 24.1 | 10 |
| 第90天 | 100 | 2.31 | 38.7 | 7 |
| 第180天 | 100 | 2.28 | 72.3 | 4 |
| 第365天 | 100 | 2.25 | 145.6 | 2 |

**观察**：
- 每日复习量从 45 张逐步降至 2 张，符合间隔重复的收敛特性
- EF 缓慢下降反映随机评分中包含 Hard/Again 的累积影响
- 100张卡在一年后每日仅需复习约2张，学习效率显著提升

### 6.2 实施建议

1. **MVP 采用当前实现**：SM-2 核心算法代码已验证可用，可直接集成
2. **参数微调空间**：Hard 的间隔系数（0.6）和 Easy 的间隔系数（1.3）可在用户测试后根据实际反馈调整
3. **后续优化方向**：
   - 考虑引入 FSRS（Free Spaced Repetition Scheduler）作为高级算法选项
   - 基于用户学习数据做个性化 EF 初始值推荐
   - 添加"学习阶段"概念（新卡多次展示后才进入复习阶段）
