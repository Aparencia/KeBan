## 一、AI 功能需求梳理

基于确认版需求列表（MVP-2），AI 接口层需支撑以下四大核心 AI 功能：

### 1.1 笔记智能摘要（Summarize）

- **触发场景**：用户在笔记页面点击"AI 摘要"按钮
- **输入**：当前笔记全文内容（纯文本/Markdown）
- **输出**：
  - 核心要点列表（3-5 条）
  - 一句话摘要（≤50字）
  - 关键词提取（5-10 个）
- **性能要求**：响应时间 < 5s，支持流式输出（逐字展示）
- **关联需求**：REQ-022（SRS-AI-001）

### 1.2 闪卡自动生成（Generate Cards）

- **触发场景**：用户在笔记页面点击"生成闪卡"或费曼学习后自动生成
- **输入**：笔记内容 / 费曼解释文本 + 目标牌组
- **输出**：
  - 闪卡列表（正面问题 + 反面答案）
  - 每张卡片含难度预估（Easy/Medium/Hard）
  - 卡片类型标注（概念卡/事实卡/关系卡）
- **性能要求**：单次生成 ≤20 张卡片，响应时间 < 10s
- **关联需求**：REQ-023（SRS-AI-002）

### 1.3 费曼解释质量评估（Evaluate Explanation）

- **触发场景**：费曼学习第二步"教授解释"完成后点击"AI 评估"
- **输入**：
  - 原始概念描述
  - 用户的费曼解释文本
- **输出**：
  - 总体评分（1-10 分）
  - 准确性评分（是否正确传达概念）
  - 简洁性评分（是否用简单语言）
  - 完整性评分（是否覆盖关键点）
  - 改进建议（具体文字建议 2-3 条）
  - 薄弱点标记（与原文对比，标注遗漏/错误部分）
- **性能要求**：响应时间 < 8s
- **关联需求**：REQ-024（SRS-AI-003）

### 1.4 番茄钟智能推荐（Recommend Duration）

- **触发场景**：番茄钟启动前，基于用户历史数据推荐配置
- **输入**：
  - 用户历史番茄钟记录（最近30天）
  - 当前时间段（上午/下午/晚上）
  - 当前科目标签
- **输出**：
  - 推荐番茄时长（分钟）
  - 推荐短休/长休时长
  - 推荐理由（基于数据分析的简短说明）
- **性能要求**：响应时间 < 3s
- **关联需求**：REQ-025（SRS-AI-004）

## 二、接口设计方案

### 2.1 核心接口定义

```typescript
/**
 * AI 插件统一接口
 * 所有 AI 功能均通过此接口调用，屏蔽底层供应商差异
 */
interface AIPlugin {
  /** 检查 AI 服务是否可用 */
  isAvailable(): Promise<boolean>;

  /** 获取当前插件名称和版本信息 */
  getInfo(): AIPluginInfo;

  /** 笔记智能摘要 */
  summarize(text: string, options?: SummarizeOptions): Promise<SummarizeResult>;

  /** 从笔记自动生成闪卡 */
  generateCards(note: string, options?: CardGenOptions): Promise<FlashCard[]>;

  /** 评估费曼解释质量 */
  evaluateExplanation(text: string, concept: string): Promise<EvaluationResult>;

  /** 基于历史数据推荐番茄钟时长 */
  recommendDuration(history: PomodoroSession[]): Promise<DurationConfig>;
}

/** 插件元信息 */
interface AIPluginInfo {
  name: string;        // 插件名称，如 "GPT-4o", "Qwen2.5"
  version: string;     // 版本号
  capabilities: AICapability[];  // 支持的能力列表
  limits: {
    maxInputTokens: number;   // 最大输入 token 数
    maxOutputTokens: number;  // 最大输出 token 数
    rateLimit: number;        // 每分钟最大请求数
  };
}

/** AI 能力枚举 */
enum AICapability {
  SUMMARIZE = 'summarize',
  GENERATE_CARDS = 'generateCards',
  EVALUATE_EXPLANATION = 'evaluateExplanation',
  RECOMMEND_DURATION = 'recommendDuration',
}
```

/**
 * 云端 AI 插件基类
 * 封装通用的 HTTP 调用、重试、限流逻辑
 */
abstract class CloudAIPlugin implements AIPlugin {
  protected apiKey: string;
  protected endpoint: string;
  protected model: string;
  protected retryPolicy: RetryPolicy;
  protected rateLimiter: RateLimiter;

  constructor(config: CloudAIConfig) {
    this.apiKey = config.apiKey;
    this.endpoint = config.endpoint;
    this.model = config.model;
    this.retryPolicy = new RetryPolicy({ maxRetries: 3, backoffMs: 1000 });
    this.rateLimiter = new RateLimiter({ requestsPerMinute: 60 });
  }

  async isAvailable(): Promise<boolean> {
    try {
      await this.healthCheck();
      return true;
    } catch {
      return false;
    }
  }

  protected abstract healthCheck(): Promise<void>;
  protected abstract callModel(prompt: string, system?: string): Promise<string>;
}

 * 本地降级插件
 * AI 服务不可用时的降级方案

class LocalFallbackPlugin implements AIPlugin {
  getInfo(): AIPluginInfo {
    return {
      name: 'Local Fallback',
      version: '1.0.0',
      capabilities: [],  // 降级模式不支持任何 AI 能力
      limits: { maxInputTokens: 0, maxOutputTokens: 0, rateLimit: 0 },
    };
  }

  isAvailable(): Promise<boolean> {
    return Promise.resolve(true);  // 降级插件始终可用
  }

  async summarize(): Promise<SummarizeResult> {
    throw new AIFeatureUnavailableError('AI 摘要功能需要网络连接，当前处于离线模式');
  }

  async generateCards(): Promise<FlashCard[]> {
    throw new AIFeatureUnavailableError('AI 生成闪卡功能需要网络连接，当前处于离线模式');
  }

  async evaluateExplanation(): Promise<EvaluationResult> {
    throw new AIFeatureUnavailableError('AI 费曼评估功能需要网络连接，当前处于离线模式');
  }

  async recommendDuration(history: PomodoroSession[]): Promise<DurationConfig> {
    // 番茄钟推荐可基于本地历史数据计算，无需网络
    return this.calculateLocalRecommendation(history);
  }

  /**
   * 基于本地历史数据的简单推荐算法
   * 计算最近 7 天同时段的平均完成时长
   */
  private calculateLocalRecommendation(history: PomodoroSession[]): DurationConfig {
    const recent = history.filter(s => {
      const daysDiff = (Date.now() - s.startTime.getTime()) / (1000 * 60 * 60 * 24);
      return daysDiff <= 7 && s.completed;
    });

    const avgDuration = recent.length > 0
      ? Math.round(recent.reduce((sum, s) => sum + s.duration, 0) / recent.length)
      : 25;  // 默认 25 分钟

    return {
      focusMinutes: Math.max(15, Math.min(45, avgDuration)),
      shortBreakMinutes: 5,
      longBreakMinutes: 15,
      longBreakInterval: 4,
      reasoning: `基于最近 7 天 ${recent.length} 次学习记录的平均值推荐`,
    };
  }
}

### 2.4 插件管理器

```typescript
/**
 * AI 插件管理器
 * 负责插件注册、选择、切换和生命周期管理
 */
class AIPluginManager {
  private plugins: Map<string, AIPlugin> = new Map();
  private activePluginId: string | null = null;
  private fallbackPlugin: LocalFallbackPlugin = new LocalFallbackPlugin();

  /** 注册 AI 插件 */
  register(id: string, plugin: AIPlugin): void {
    this.plugins.set(id, plugin);
  }

  /** 设置当前活跃插件 */
  async setActive(id: string): Promise<void> {
    const plugin = this.plugins.get(id);
    if (!plugin) throw new Error(`插件 ${id} 未注册`);
    if (await plugin.isAvailable()) {
      this.activePluginId = id;
    } else {
      console.warn(`插件 ${id} 不可用，保持当前插件`);
    }
  }

  /** 获取当前可用插件，不可用时返回降级插件 */
  async getPlugin(): Promise<AIPlugin> {
    if (this.activePluginId) {
      const plugin = this.plugins.get(this.activePluginId)!;
      if (await plugin.isAvailable()) return plugin;
    }
    // 尝试找到任意可用插件
    for (const [id, plugin] of this.plugins) {
      if (await plugin.isAvailable()) {
        this.activePluginId = id;
        return plugin;
      }
    }
    // 全部不可用，返回降级插件
    return this.fallbackPlugin;
  }

  /** 健康检查所有已注册插件 */
  async healthCheckAll(): Promise<Map<string, boolean>> {
    const results = new Map<string, boolean>();
    for (const [id, plugin] of this.plugins) {
      results.set(id, await plugin.isAvailable());
    }
    return results;
  }
}

## 三、降级策略原型验证

### 3.1 AI 可用时的调用流程

```
用户操作 → UI 层（Loading 状态）
         → AIFacade（统一入口）
         → AIPluginManager.getPlugin()
         → CloudAIPlugin（具体供应商实现）
           → 构建 Prompt
           → RateLimiter.check()
           → HTTP Request（含 retry）
           → 解析 Response
         → 结果缓存（LRU Cache，TTL=5min）
         → UI 层展示结果
```

### 3.2 AI 不可用时的降级行为

| AI 功能 | 降级行为 | 用户提示 |
|---------|---------|---------|
| 笔记摘要 | 提取前200字 + 标题列表作为简易摘要 | "AI 摘要暂时不可用，已为您生成简易摘要" |
| 闪卡生成 | 禁用"AI 生成"按钮，引导手动创建 | "AI 生成暂不可用，请手动创建闪卡" |
| 费曼评估 | 禁用"AI 评估"按钮，保留手动自评 | "AI 评估暂不可用，请先完成自评" |
| 番茄钟推荐 | 切换为本地统计算法（基于历史均值） | "已使用本地数据为您推荐时长" |

### 3.3 网络断开/恢复时的切换测试

#### 测试场景 1：网络断开

```
前置：AI 插件正常工作
操作：断开网络
预期：
  1. AIPluginManager 检测到 CloudAIPlugin 不可用
  2. 自动切换到 LocalFallbackPlugin
  3. UI 显示"离线模式"标识
  4. 番茄钟推荐仍可正常使用（本地算法）
  5. 其他 AI 功能显示友好提示
```

#### 测试场景 2：网络恢复

```
前置：处于离线模式
操作：恢复网络
预期：
  1. AIPluginManager 周期性健康检查（每30s）发现云端恢复
  2. 自动切换回 CloudAIPlugin
  3. UI 移除"离线模式"标识
  4. 所有 AI 功能恢复正常
```

#### 测试场景 3：网络抖动

```
前置：AI 插件正常工作
操作：模拟间歇性断网（每5s断开2s）
预期：
  1. RetryPolicy 生效，自动重试（指数退避）
  2. 3次重试仍失败后切换到降级模式
  3. 不出现卡死/崩溃/无限 Loading
```

### 4.2 各供应商详细分析

#### GPT-4o（OpenAI）

- **优势**：综合能力最强，指令遵循度高，生成的闪卡质量优秀
- **劣势**：价格较高，需要海外 API 访问，可能受网络限制
- **适用场景**：对 AI 质量要求最高的核心功能
- **接入复杂度**：中等（需要 API Key 管理和代理）

#### Claude 3.5 Sonnet（Anthropic）

- **优势**：长文本理解能力强，费曼评估准确度最高，输出结构化程度好
- **劣势**：中文表达偶有不自然，价格与 GPT-4o 接近
- **适用场景**：长笔记摘要、复杂概念评估
- **接入复杂度**：中等

#### 通义千问 Max（阿里云）

- **优势**：中文支持最佳，价格低廉，国内访问稳定，合规性好
- **劣势**：复杂推理能力稍弱，卡片生成质量一般
- **适用场景**：作为主力供应商，承担大部分日常 AI 调用
- **接入复杂度**：低（阿里云生态，直接对接）

#### DeepSeek-V2（深度求索）

- **优势**：性价比极高，中文理解能力好，API 稳定
- **劣势**：生态较新，社区支持有限
- **适用场景**：成本敏感场景的替代方案
- **接入复杂度**：低

### 4.3 推荐策略：多供应商架构

```
                    ┌──────────────────┐
                    │  AIPluginManager │
                    └────────┬─────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
    ┌─────────▼──────┐ ┌────▼────────┐ ┌───▼─────────────┐
    │ 通义千问(主力)  │ │ GPT-4o(备选) │ │ LocalFallback   │

    └────────────────┘ └─────────────┘ └─────────────────┘

**路由策略**：
1. 默认使用通义千问（成本低，国内稳定）
2. 费曼评估等复杂场景可切换到 GPT-4o/Claude
3. 全部云端不可用时降级到 LocalFallback
4. 支持用户手动切换供应商（高级设置）

## 六、结论

### 6.1 核心结论

1. **插件化架构可行**：通过统一的 `AIPlugin` 接口，可以屏蔽不同 AI 供应商的差异，实现灵活切换
2. **推荐多供应商策略**：通义千问作为主力（成本低、中文好），GPT-4o 作为高质量备选
3. **降级策略完善**：`LocalFallbackPlugin` 确保离线场景下番茄钟推荐仍可用，其他功能优雅降级
4. **成本可控**：混合方案下每用户月成本约 ¥5，通过缓存和限流可进一步控制
