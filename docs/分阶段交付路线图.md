# 课伴分阶段交付路线图

本文档定义课伴项目从启动到上线及后续演进的完整交付路线图，采用两阶段交付策略，确保价值尽早交付。

---

## 一、两阶段交付策略概述

### 1.1 为什么采用两阶段交付

一次性交付"完美产品"的风险极高——工期长、需求漂移、市场变化。两阶段交付将风险分散，并在每个阶段都向用户交付可感知的价值：

- **MVP-1**（第 28 周发布）：纯本地学习工具。用户无需注册、无需联网即可使用全部核心功能（番茄钟、智能笔记、闪卡、费曼学习法）。**先占领本地学习工具细分市场，建立用户基础和口碑。**
- **MVP-2**（第 36 周发布）：叠加云同步 + AI 增强能力，作为增值功能上线。**在已有用户基础上提升留存和付费转化。**

### 1.2 两阶段交付的核心原则

| 原则 | 说明 |
|-----|------|
| 价值前置 | MVP-1 独立可用，不依赖 MVP-2 的任何功能 |
| 风险分散 | 技术风险（存储、同步）在 MVP-1 中提前验证 |
| 快速验证 | MVP-1 发布后立即收集用户反馈，指导 MVP-2 优先级 |
| 渐进复杂 | 先做简单的本地功能，再做复杂的云端/AI 功能 |

## 三、各阶段详细交付内容

### 3.1 MVP-1（W28）—— 本地学习工具

**目标**：交付一个完全离线可用、无需注册的本地学习工具，覆盖四大核心学习模块。

#### 番茄钟模块

| 功能 | 说明 |
|-----|------|
| 基础计时 | 25/5 分钟标准番茄钟，支持自定义时长 |
| 上课模式 | 适配课堂场景，45/10 分钟模式 |
| 自习模式 | 适配自主学习，自由配置时间 |
| 专注统计 | 每日/每周/每月专注时长统计图表 |
| 个性化配置 | 提示音、主题色、自动开始等设置 |

#### 智能笔记模块

| 功能 | 说明 |
|-----|------|
| 富文本编辑器 | 支持标题、列表、代码块、公式、图片等 |
| 模板系统 | 预设笔记模板（课堂笔记、读书笔记、会议纪要等） |
| 全文搜索 | 基于 IndexedDB 的本地全文搜索 |
| 笔记组织 | 文件夹 + 标签双维度组织 |
| 导入导出 | 支持 Markdown、HTML、PDF 格式 |

#### 闪卡模块

| 功能 | 说明 |
|-----|------|
| CRUD 操作 | 创建、编辑、删除、归档闪卡 |
| SM-2 算法 | 基于间隔重复算法安排复习计划 |
| 学习会话 | 开始学习 → 评估掌握度 → 安排下次复习 |
| 牌组管理 | 按主题组织闪卡为牌组 |
| 学习统计 | 每日学习数量、掌握率、遗忘曲线 |

#### 费曼学习法模块

| 功能 | 说明 |
|-----|------|
| 选择概念 | 从笔记或手动输入中选择要学习的概念 |
| 用自己的话解释 | 提供编辑器让用户用自己的语言解释概念 |
| 发现知识缺口 | 标记不确定的部分，形成待补强清单 |
| 回顾与简化 | 回顾历史解释，逐步精炼理解 |
| 记录与追踪 | 费曼学习记录持久化，进度可视化 |

#### 基础设施

| 功能 | 说明 |
|-----|------|
| 本地存储 | IndexedDB + Dexie.js，数据持久化 |
| 导入导出 | 全量数据备份与恢复（JSON 格式） |
| 基础本地账户 | 本地用户标识，无需联网注册 |
| 学习数据看板 | 汇总各模块学习数据，统一展示 |
| PWA 支持 | 离线可用、安装到桌面、推送通知 |

#### MVP-1 交付标准

- 所有核心功能在 Chrome、Safari、Firefox、Edge 最新版上正常运行
- 完全离线环境下核心功能可用
- 首屏加载时间 ≤ 3 秒（4G 网络）
- 无 Blocker 和 Critical 级别 Bug
- 测试覆盖率 ≥ 75%

### 3.2 MVP-2（W36）—— 云同步 + AI 增强

**目标**：在 MVP-1 基础上叠加云同步能力和 AI 增强功能，提升用户体验和产品商业价值。

#### 云同步引擎

| 功能 | 说明 |
|-----|------|
| 同步核心 | LWW + 版本向量实现多设备数据同步 |
| 冲突处理 | 冲突检测 + 副本保留 + 用户手动选择 |
| 增量同步 | 仅同步变更数据，节省带宽和流量 |
| 同步状态 | 实时展示同步进度和状态 |
| 选择性同步 | 用户可选择同步哪些数据类型 |

#### AI 增强功能

| 功能 | 说明 |
|-----|------|
| AI 笔记摘要 | 一键生成笔记摘要，提炼核心要点 |
| AI 笔记续写 | 基于已有内容智能续写，辅助思考 |
| AI 自动生成闪卡 | 从笔记内容自动提取知识点生成闪卡 |
| AI 费曼评估 | 评估用户的费曼解释质量，给出改进建议 |
| AI 番茄钟推荐 | 根据学习数据推荐最优专注时长 |

#### 双模式框架

| 功能 | 说明 |
|-----|------|
| 功能级按需激活 | AI 功能点击时触发联网，其余时间本地运行 |
| 在线状态指示 | 清晰展示当前网络状态和可用功能 |
| 离线降级 | 网络不可用时 AI 功能优雅降级，不阻断核心流程 |

#### 云用户系统

| 功能 | 说明 |
|-----|------|
| 注册/登录 | 邮箱 + 密码注册，支持第三方登录 |
| 数据云存储 | 用户数据安全存储于云端 |
| 账户管理 | 密码重置、账户设置、数据删除 |

#### MVP-2 交付标准

- 双设备同步数据一致性 ≥ 99.5%
- AI 功能响应时间 ≤ 3 秒
- AI 生成内容准确率 ≥ 85%
- 无 Blocker 和 Critical 级别 Bug
- 测试覆盖率 ≥ 80%

### 3.3 Beta（W44）—— 公测验证

**目标**：面向真实用户开放，收集大规模使用反馈，发现并修复问题。

| 交付项 | 详细内容 |
|-------|---------|
| 全功能集成 | MVP-1 + MVP-2 全部功能集成，端到端流程打通 |
| 全面测试 | 功能测试、兼容性测试、性能测试、安全测试、可用性测试 |
| 性能优化 | 首屏加载优化、大数据量性能优化、内存泄漏修复 |
| Bug 修复 | 修复 Alpha 阶段和内测发现的所有 Blocker/Critical Bug |
| 用户引导 | 完善新用户引导流程、帮助文档 |
| 监控体系 | 错误监控、性能监控、用户行为分析 |
| 反馈渠道 | 应用内反馈入口、用户社区搭建 |

#### Beta 交付标准

- 所有已知 Blocker/Critical Bug 已修复
- 崩溃率 ≤ 0.1%
- 用户满意度评分 ≥ 4.0/5.0
- 核心流程转化率 ≥ 70%

## 附录：关键假设与依赖

### 关键假设

1. 团队在 W01 前完成组建，核心岗位（前端、后端、产品、设计）到位
2. 本地存储方案在 M1.5 评审中顺利通过，无重大技术障碍
3. AI 供应商 API 在 MVP-2 开发期间保持稳定可用
4. MVP-1 发布后能获得足够的早期用户反馈来指导 MVP-2 优先级

| 依赖项 | 提供方 | 影响阶段 | 风险等级 |
