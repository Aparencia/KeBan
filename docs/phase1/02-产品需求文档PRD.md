## 一、产品概述

课伴（KeBan）是一款面向网课学生的本地优先学习辅助工具，集成番茄钟、智能笔记、闪卡、费曼学习法四大核心模块，辅以 AI 增强能力，帮助学生提升学习专注力、知识管理效率与记忆巩固效果。

产品遵循 **"本地优先、AI 增强"** 的核心原则：所有基础功能在纯离线环境下完整可用，AI 功能作为在线增值层叠加，绝不因网络状况影响核心学习体验。

## 二、🍅 番茄钟模块（Pomodoro Timer）

### 2.1 模块概述与定位

番茄钟是课伴的"时间感知层"，帮助学生量化学习投入、培养专注习惯。它不只是一个简单的计时器，而是一个适配网课场景的智能时间管理工具，核心区分"上课模式"与"自习模式"两大场景。

**定位关键词**：专注计时 · 场景适配 · 数据量化

### 2.2 本地模式功能清单

| # | 功能项 | 描述 |
|---|--------|------|
| F-PT-01 | 核心计时引擎 | 工作 / 短休息 / 长休息三段式计时，精确到秒 |
| F-PT-02 | 上课模式 | 不弹出休息提醒，计时结束后静默标记，适合 40-45 分钟网课 |
| F-PT-03 | 自习模式 | 标准 25/5 分钟节奏，每 4 个番茄后 15 分钟长休息 |
| F-PT-04 | 通知提醒系统 | Notification API 浏览器通知 + 声音提示 + 视觉动画（进度环变色） |
| F-PT-05 | 每日统计 | 今日专注时长、完成番茄数、最长连续记录 |
| F-PT-06 | 每周统计图表 | 柱状图展示每日时长 + 7 日移动平均趋势线 |
| F-PT-07 | 自定义配置 | 工作/短休息/长休息时长、每轮番茄数、提示音类型均可自定义 |
| F-PT-08 | 本地持久化 | 所有计时记录和配置存入 IndexedDB，关闭浏览器不丢失 |
| F-PT-09 | 后台稳定运行 | 基于 Web Worker 计时，切换标签页/最小化不中断 |
| F-PT-10 | 日期范围筛选 | 统计页面支持自定义日期范围查看历史数据 |

### 2.3 AI 增强功能清单

| # | 功能项 | 描述 | 降级策略 |
|---|--------|------|---------|
| F-PT-AI-01 | 智能时长推荐 | 基于历史数据分析用户专注规律，推荐最佳工作时长 | 退回默认 25 分钟 |
| F-PT-AI-02 | 专注度预测 | 根据时间段和历史数据预测当前时段专注度（高/中/低） | 隐藏预测指标 |

### 2.4 交互设计要点

- **大数字倒计时**：界面中央以超大字号显示 "MM:SS" 倒计时，3 米外清晰可读
- **进度环动画**：倒计时外围环形进度条，颜色随模式变化（工作=红色、短休息=绿色、长休息=蓝色）
- **模式切换入口**：顶部 Tab 栏或右上角开关，切换时有明确视觉反馈
- **计时中极简 UI**：计时进行中自动隐藏非必要 UI 元素（导航栏、统计入口），仅保留倒计时+暂停/停止按钮
- **浮动窗口（远期）**：支持小窗模式，计时器悬浮在其他应用上方

### 2.5 应用场景

**场景 A：网课 45 分钟静默计时**  
小明下午有 3 节网课，每节 45 分钟。上课前切换到"上课模式"，点击启动，计时器静默运行 45 分钟。期间无任何弹窗或声音干扰。下课后小明查看统计，看到今天已专注 135 分钟。

**场景 B：晚自习 25/5 节奏学习**  
小红晚上在家自习，切换到"自习模式"，按下启动键。25 分钟后轻柔提示音响起，屏幕显示"休息一下"动画。5 分钟休息后自动进入下一个 25 分钟工作时段。4 个番茄后进入 15 分钟长休息。

## 三、📝 智能笔记模块（Smart Notes）

### 3.1 模块概述与定位

智能笔记是课伴的"知识沉淀层"，提供从课堂速记到课后整理的完整笔记工作流。内置多种经典笔记模板，配合本地全文搜索，构建学生的个人知识库。AI 增强层可实现智能摘要、续写润色等高级功能。

**定位关键词**：结构化记录 · 知识组织 · 智能整理

### 3.2 本地模式功能清单

| # | 功能项 | 描述 |
|---|--------|------|
| F-NT-01 | 富文本编辑器 | 支持标题（H1-H6）、有序/无序列表、代码块、表格、图片插入、加粗/斜体/下划线 |
| F-NT-02 | 大纲式模板 | 多级缩进结构，支持折叠/展开子节点 |
| F-NT-03 | 康奈尔笔记模板 | 三栏布局：线索栏（左）+ 笔记栏（右）+ 总结栏（底部） |
| F-NT-04 | 思维导图式模板 | 中心主题 + 分支节点，支持拖拽调整层级 |
| F-NT-05 | 自由笔记模板 | 无预设结构，完全自由排版 |
| F-NT-06 | 课堂速记模板 | 时间戳自动标注 + 快速要点录入格式 |
| F-NT-07 | 文件夹组织 | 支持最多 3 级嵌套文件夹，拖拽移动笔记 |
| F-NT-08 | 标签系统 | 自由创建标签、笔记可多标签、按标签筛选 |
| F-NT-09 | 全文搜索 | 本地全文索引，1000 篇笔记规模下搜索响应 < 500ms |
| F-NT-10 | 导入导出 | 支持 Markdown / HTML / JSON 格式导入导出 |
| F-NT-11 | 自动保存 | 编辑内容自动保存（间隔 ≤ 10 秒），意外关闭不丢失 |
| F-NT-12 | 版本历史 | 本地保存最近 10 个版本，可回滚 |

### 3.3 AI 增强功能清单

| # | 功能项 | 描述 | 降级策略 |
|---|--------|------|---------|
| F-NT-AI-01 | 智能摘要 | 一键生成笔记摘要（200-300 字），覆盖核心要点 | 按钮置灰+提示"AI 暂不可用" |
| F-NT-AI-02 | 续写/润色 | 选中段落可请求 AI 续写或润色，改善表达 | 隐藏 AI 工具栏按钮 |
| F-NT-AI-03 | 关键词提取高亮 | 自动识别笔记关键词并在正文中高亮标注 | 隐藏关键词侧栏 |
| F-NT-AI-04 | 语义搜索 | 基于向量数据库的语义相似度搜索（非关键词匹配） | 回退到本地关键词搜索 |

### 3.4 交互设计要点

- **三栏布局**：左侧导航（文件夹/标签树）+ 中间列表（笔记卡片化预览）+ 右侧编辑区
- **模板选择器**：新建笔记时弹出模板选择浮层，每个模板带预览缩略图和适用场景说明
- **卡片化预览**：列表页每篇笔记以卡片形式展示，显示标题、创建时间、标签、内容前 3 行预览
- **搜索快捷键**：`Ctrl/Cmd + K` 全局唤起搜索框，输入即搜，实时显示结果
- **分屏对照**：支持笔记分屏，左边看笔记右边写总结（适配康奈尔模板复习场景）

### 3.5 应用场景

**场景 A：上课选康奈尔模板边听边记**  
小李上物理课，打开课伴新建笔记，选择"康奈尔笔记"模板。老师在讲牛顿第三定律，小李在右侧笔记栏实时记录公式和例题，左侧线索栏标注"作用力""反作用力"等关键词。课后在底部总结栏写下本节核心结论。

**场景 B：课后搜索关键词复习**  
期末考试前，小张打开课伴笔记模块，按 `Ctrl+K` 搜索"光合作用"，200ms 内找到 3 篇相关笔记。点击其中一篇，关键词"光反应""暗反应""叶绿体"已高亮显示，快速定位到需要复习的段落。

## 四、🃏 闪卡模块（Flashcards）

### 4.1 模块概述与定位

闪卡是课伴的"记忆巩固层"，基于经过科学验证的 SM-2 间隔重复算法，帮助学生以最优的时间投入实现长期记忆。支持自建卡片、牌组管理、学习会话等完整功能闭环。

**定位关键词**：间隔重复 · 科学记忆 · 知识卡片化

### 4.2 本地模式功能清单

| # | 功能项 | 描述 |
|---|--------|------|
| F-FC-01 | 卡片 CRUD | 创建/编辑/删除卡片，正面（问题）+ 背面（答案），支持文字/图片/代码块 |
| F-FC-02 | SM-2 算法引擎 | 基于 Again/Hard/Good/Easy 四档评分计算下次复习时间，维护 EF/Interval 参数 |
| F-FC-03 | 牌组管理 | 创建/编辑/删除牌组，按科目/章节分组 |
| F-FC-04 | 子牌组嵌套 | 支持至少 3 级嵌套（科目 → 章节 → 知识点） |
| F-FC-05 | 学习会话 | 混合推送新卡片和到期复习卡片，支持设定每轮数量上限 |
| F-FC-06 | 学习统计 | 今日复习数/新学数/正确率、遗忘曲线可视化、牌组热力图 |
| F-FC-07 | 批量导入 | 支持 CSV / JSON 格式批量导入卡片 |
| F-FC-08 | 卡片搜索 | 全局和牌组内搜索，检索正面和背面内容 |
| F-FC-09 | 数据持久化 | 所有卡片数据、SM-2 参数、学习记录本地持久化 |

### 4.3 AI 增强功能清单

| # | 功能项 | 描述 | 降级策略 |
|---|--------|------|---------|
| F-FC-AI-01 | AI 自动生成卡片 | 基于笔记内容自动生成 5-20 张问答卡片 | 提示手动创建+提供模板 |
| F-FC-AI-02 | 智能难度调整 | 根据用户整体表现动态调整 EF 计算参数 | 使用标准 SM-2 参数 |
| F-FC-AI-03 | 解释补充 | 复习时点击"AI 解释"获取卡片内容的补充说明 | 隐藏解释按钮 |

### 4.4 交互设计要点

- **卡片翻转动画**：点击卡片触发 3D 翻转动画（正面→背面），流畅度 ≥ 60fps
- **四评分按钮**：翻转后底部横排 4 个按钮：Again（红）/ Hard（橙）/ Good（绿）/ Easy（蓝），每个按钮上方显示预估下次间隔
- **牌组列表视图**：每个牌组以卡片行展示，显示名称/总卡片数/今日到期数/进度条
- **学习进度条**：学习会话中顶部显示"已学/总数"进度条，实时递减剩余数量
- **快捷键支持**：空格=翻转、1/2/3/4=评分，满足高频刷用户需求

### 4.5 应用场景

**场景 A：每日 20 张到期卡片复习**  
小王每天早上打开课伴闪卡模块，看到提示"今日有 20 张卡片待复习"。进入"英语词汇"牌组，逐一翻转卡片，根据回忆情况选择评分。10 分钟完成全部复习，系统自动安排下次复习日期。

**场景 B：考前批量录入单词卡片**  
期末考试前 2 周，小陈需要准备 200 个生物学术语。他准备好 CSV 文件（正面=术语，背面=定义），一键批量导入到"生物期末"牌组。系统按 SM-2 算法每天推送适量新卡片和复习卡片，2 周后覆盖率 85%。

## 五、🧠 费曼学习法模块（Feynman Method）

### 5.1 模块概述与定位

费曼学习法是课伴的"深度理解层"，将经典的费曼四步学习法（选概念→解释→识别薄弱→简化重述）数字化为引导式工具。帮助学生从"被动接收"转向"主动输出"，通过"教会别人"的方式检验真实理解。

**定位关键词**：深度理解 · 引导式流程 · 薄弱攻克

### 5.2 本地模式功能清单

| # | 功能项 | 描述 |
|---|--------|------|
| F-FM-01 | 四步流程引擎 | 选择概念 → 用自己的话解释 → 识别薄弱点 → 简化重述，严格引导 |
| F-FM-02 | 文字记录 | 大面积文字编辑区，支持加粗/列表/代码块排版 |
| F-FM-03 | 薄弱点标记 | 选中文字后一键标记为薄弱点，高亮显示 |
| F-FM-04 | 待补强列表 | 所有薄弱点自动汇总，可按科目/日期筛选 |
| F-FM-05 | 进度追踪 | 记录每个费曼学习的完成状态和理解深度自评（1-5 星） |
| F-FM-06 | 步骤导航 | 步骤进度条可点击，支持前进/后退，已填内容不丢失 |
| F-FM-07 | 自动保存 | 每步内容自动保存（间隔 ≤ 30 秒），意外关闭可恢复 |
| F-FM-08 | 历史记录回顾 | 已完成的费曼学习归入历史列表，可随时回看和修改 |
| F-FM-09 | 薄弱点状态管理 | 薄弱点可标记"已掌握"移出待补强列表，保留历史 |
| F-FM-10 | 薄弱点统计 | 展示薄弱点总数、科目分布、本周新增/已攻克数 |

### 5.3 AI 增强功能清单

| # | 功能项 | 描述 | 降级策略 |
|---|--------|------|---------|
| F-FM-AI-01 | 解释质量评估 | AI 评估用户解释的准确性和完整性，给出改进建议 | 隐藏评估面板 |
| F-FM-AI-02 | 引导问题生成 | 根据所选概念生成引导性问题，帮助用户展开解释 | 显示空白编辑区+提示自行思考 |
| F-FM-AI-03 | 录音转文字 | 用户口述解释，AI 转写为文字自动填入 | 仅提供文字输入方式 |

### 5.4 交互设计要点

- **步骤进度条**：顶部横向 4 步进度条，当前步骤高亮放大，已完成步骤打勾可点击回看
- **大面积编辑区**：步骤 2 和步骤 4 的核心区域为大面积文字编辑器，占页面 60% 以上
- **标记薄弱按钮**：选中文字后浮现"标记薄弱"按钮（类似富文本工具栏），点击后文字背景变为黄色高亮
- **右侧抽屉面板**：薄弱点列表以右侧抽屉形式展开，不遮挡主编辑区
- **完成庆祝动画**：四步全部完成时播放简短庆祝动画（彩带/徽章），正向激励

### 5.5 应用场景

**场景 A：学习牛顿第三定律，标记薄弱点**  
小周在物理课学了牛顿第三定律，打开费曼学习法模块。步骤 1 输入"牛顿第三定律"；步骤 2 用自己的话写下解释；步骤 3 发现自己对"作用力与反作用力为什么不抵消"解释不清，标记为薄弱点；步骤 4 尝试用更简洁的语言重新表述。完成后该薄弱点进入"待补强"列表，周末集中攻克。

**场景 B：考前浏览待补强列表**  
期末考试前，小刘打开费曼模块的"待补强"列表，看到本学期积累了 15 个薄弱点，按科目分类显示。重点复习其中标记为"高优先级"的 5 个，逐一点击跳转回对应的费曼学习记录重新理解，理解后标记"已掌握"。

## 六、模块间协同关系

```
                    ┌───────────────┐
                    │   🍅 番茄钟    │
                    │  （时间感知层）  │
                    └───────┬───────┘
                            │ 记录学习时段
                            ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│  📝 智能笔记   │───▶│  🃏 闪卡       │◀───│ 🧠 费曼学习法  │
│ （知识沉淀层）  │    │ （记忆巩固层）  │    │（深度理解层）  │
└───────────────┘    └───────────────┘    └───────────────┘
    笔记内容 ──────▶ AI生成卡片              薄弱点归入笔记
```

- **笔记 → 闪卡**：笔记内容可通过 AI 自动转化为闪卡，形成"记录 → 记忆"闭环
- **费曼 → 笔记**：费曼学习中的薄弱点可关联到对应笔记，便于定向复习
- **番茄钟 → 全局**：番茄钟记录的学习时长为各模块提供时间维度数据

## 七、非功能需求摘要

| 维度 | 要求 |
|------|------|
| 首屏加载 | < 3 秒（生产环境） |
| 操作响应 | < 200ms（所有交互操作） |
| 离线可用性 | 100% 基础功能纯离线可用 |
| 数据安全 | 本地数据加密存储（AES-256） |
| 浏览器兼容 | Chrome / Firefox / Edge / Safari 最新 2 个版本 |
| 响应式设计 | 支持 1024px 以上屏幕，优先桌面端体验 |
